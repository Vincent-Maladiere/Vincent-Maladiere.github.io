13.6.1 Radial basis function networks

Consider a 1-layer neural network where the hidden units are given by:
ϕ(x)=[K(x,μ1),…,K(x,μK)]\phi(\bold{x})=[\mathcal{K}(\bold{x},\mu_1),\dots,\mathcal{K}(\bold{x},\mu_K)]ϕ(x)=[K(x,μ1​),…,K(x,μK​)]
where μk∈X\mu_k\in\mathcal{X}μk​∈X﻿ are a set of KKK﻿ centroids, and K(x,μk)≥0\mathcal{K}(\bold{x},\mu_k)\geq 0K(x,μk​)≥0﻿ is a kernel function (see section 17.1).
We use the simplest kernel function, the Gaussian kernel:
Kgauss(x,y)=exp⁡(−12σ2∣∣x−y∣∣22)\mathcal{K}_{gauss}(\bold{x,y})=\exp\Big(-\frac{1}{2\sigma^2}{||\bold{x-y}||^2_2} \Big)Kgauss​(x,y)=exp(−2σ21​∣∣x−y∣∣22​)
where σ\sigmaσ﻿ is called the bandwith. This is called a radial basis function (RBF) kernel.
A RBF network has the form:
p(y∣x;θ)=p(y∣w⊤ϕ(x))p(y|x;\theta)=p(y|\bold{w}^\top \phi(\bold{x}))p(y∣x;θ)=p(y∣w⊤ϕ(x))
where θ=(μ,w)\theta=(\mu,\bold{w})θ=(μ,w)﻿. The centroids can be fixed or learned using unsupervised learning methods like K-means.
Alternatively, we can associate one centroid per datapoint, μk=xn\mu_k=x_nμk​=xn​﻿ with K=NK=NK=N﻿. This is an example of non-parametric model, since the number of parameters grow with the size of the data.
If K=NK=NK=N﻿ the model can perfectly interpolate the data and overfit. However, by ensuring the weights w\bold{w}w﻿ are sparse, the model will use a finite number of weights and will result in a sparse kernel machine.
Another way to avoid overfitting is to use a Bayesian approach, by marginalizing the weights, this gives rise to the Gaussian process model.
13.6.1.1 RBF network for regression
We can use RBF for regression by defining:
p(y∣x;θ)=N(y∣w⊤ϕ(x),σ2)p(y|x;\theta)=\mathcal{N}(y|\bold{w}^\top \phi(\bold{x}),\sigma^2)p(y∣x;θ)=N(y∣w⊤ϕ(x),σ2)
Below, we fit 1d data to a model with K=10K=10K=10﻿ uniformly spaced RBF clusters, with the bandwith ranging from low to large.
When the bandwidth is small, the predicted function is wiggly since points far from the clusters will lead to predicted value of 0.
On the contrary, if the bandwidth is large, each points is equally close to all clusters, so the prediction is a straight line.
13.6.1.2 RBF network for classification
We can use RBF in classification by considering:
p(y∣x;θ)=Ber(y∣σ(w⊤ϕ(x)))p(y|x;\theta)=\mathrm{Ber}(y|\sigma(\bold{w}^\top \phi(\bold{x})))p(y∣x;θ)=Ber(y∣σ(w⊤ϕ(x)))
RBF are able to solve the XOR problem.

13.6.2 Mixtures of experts

When considering regression, it is common to assume an unimodal output distribution. However, this won’t work well with one-to-many functions, in which each input is mapped to multiple outputs.
There are many real-world problems like 3d pose estimation of a person given an image, colorization of a black and white image, and predicting future frames of a video sequence.
Any model trained to maximize likelihood using a unimodal output density (even non-linear and flexible ones) will have poor performances on one-to-many functions since it will produce a blurry average output.
To prevent this problem of regression to the mean, we can use a conditional mixture model. We assume the output is a weighted mixture of KKK﻿ different outputs, corresponding to different modes of the output distribution for each input x\bold{x}x﻿.
In the Gaussian case, this becomes:
p(y∣x)=∑k=1Kp(y∣x,z=k)p(z=k∣x)p(\bold{y}|\bold{x})=\sum_{k=1}^K p(\bold{y}|\bold{x},z=k)p(z=k|\bold{x})p(y∣x)=k=1∑K​p(y∣x,z=k)p(z=k∣x)
where:
p(y∣x,z=k)=N(y∣fμ,k(x),diag(fσ,k(x)))p(z=k∣x)=Cat(z∣Sk(fz(x)))\begin{align} p(\bold{y}|\bold{x},z=k)&=\mathcal{N}(\bold{y}|f_{\mu,k}(\bold{x}),\mathrm{diag}( f_{\sigma,k}(\bold{x})))\\ p(z=k|\bold{x}) &=\mathrm{Cat}(z|\mathcal{S}_k(f_z(\bold{x}))) \end{align}p(y∣x,z=k)p(z=k∣x)​=N(y∣fμ,k​(x),diag(fσ,k​(x)))=Cat(z∣Sk​(fz​(x)))​​
Here, fzf_zfz​﻿ predicts which mixture to use, fμ,kf_{\mu,k}fμ,k​﻿ and fσ,kf_{\sigma, k}fσ,k​﻿ predict the mean and the variance of the kth Gaussian.
This is called a Mixture of Expert (MOE), where we choose submodels that are experts on some region of the input space, with a gating function p(z=k∣x)p(z=k|\bold{x})p(z=k∣x)﻿.
By picking the most likely model expert for a given x\bold{x}x﻿, we can activate a subset of the model. This is an example of conditional computation.

13.6.2.2 Mixture density networks
The gating and expert functions can be any kind of conditional probabilistic model, like DNN. In this situation, the model becomes a mixture density network (MDN).
13.6.2.3 Hierarchical MOEs
If each network is a MOE itself, we obtain a hierarchical mixture of experts.
An HME with LLL﻿ levels can be thought of as a “soft” decision tree of depth LLL﻿, where each example is passed through every branch of the tree, and the final prediction is the weighted average.

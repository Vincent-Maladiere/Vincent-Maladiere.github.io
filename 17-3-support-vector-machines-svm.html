17.3 Support vector machines (SVM)

We now discuss a form of (non-probabilistic) predictors for classification and regression problems which have the form:
f(x)=∑i=1NαiK(x,xi)f(\bold{x})=\sum_{i=1}^N \alpha_i \mathcal{K}(\bold{x},\bold{x}_i)f(x)=i=1∑N​αi​K(x,xi​)
By adding some constraints, we can ensure only few coefficient αi\alpha_iαi​﻿ are non-zero, so that predictions only depends on a subset of the training points, called support vectors.
The resulting model is called support vector machine (SVM).

17.3.1 Large margin classifiers

Consider a binary classifier of the form h(x)=sign(f(x))h(\bold{x})=\mathrm{sign}(f(\bold{x}))h(x)=sign(f(x))﻿, where the decision boundary is given by:
f(x)=w⊤x+w0f(\bold{x})=\bold{w}^\top \bold{x}+w_0f(x)=w⊤x+w0​
In the SVM literature, it is common to use class labels of {−1,+1}\{-1,+1\}{−1,+1}﻿ rather than {0,1}\{0,1\}{0,1}﻿. We denote such targets as y~\tilde{y}y~​﻿ instead of yyy﻿.
There might be many lines that separate the data, and intuitively we want to pick the one that has the maximum margin. This represents the distance between the closest point to the decision boundary.
The model on the left gives the best solution, because more robust to perturbation of the data.
How can we compute such a large margin classifier? First, we need to derive an expression of the distance of a point to the decision boundary.
We see that:
x=x⊥+rw∣∣w∣∣\bold{x}=\bold{x}_\bot+r\frac{\bold{w}}{||\bold{w}||}x=x⊥​+r∣∣w∣∣w​
where rrr﻿ is the distance from x\bold{x}x﻿ to the decision boundary whose normal vector is w\bold{w}w﻿, and x⊥\bold{x}_\botx⊥​﻿ is the orthogonal projection of x\bold{x}x﻿ on the decision boundary.
We would like to maximize rrr﻿, so we need to express it as a function of w\bold{w}w﻿:
f(x)=w⊤(x⊥+rw∣∣w∣∣)+w0=w⊤x⊥+w0+rw⊤w∣∣w∣∣=f(x⊥)+r∣∣w∣∣=r∣∣w∣∣\begin{align} f(\bold{x})&=\bold{w}^\top (\bold{x}_\bot+r\frac{\bold{w}}{||\bold{w}||})+w_0\\ &= \bold{w}^\top \bold{x}_\bot+w_0+r\frac{\bold{w}^\top \bold{w}}{||\bold{w}||} \\ &= f(\bold{x}_\bot)+r||\bold{w}|| \\ &= r||\bold{w}|| \end{align}f(x)​=w⊤(x⊥​+r∣∣w∣∣w​)+w0​=w⊤x⊥​+w0​+r∣∣w∣∣w⊤w​=f(x⊥​)+r∣∣w∣∣=r∣∣w∣∣​​
Since f(x⊥)=0f(\bold{x}_\bot)=0f(x⊥​)=0﻿, therefore r=f(x)/∣∣w∣∣r=f(\bold{x})/||w||r=f(x)/∣∣w∣∣﻿​
Since we want to ensure that each point is on the correct point of the boundary, we also require f(xn)y~n>0f(\bold{x}_n)\tilde{y}_n>0f(xn​)y~​n​>0﻿.
We want to maximize the distance to the closest point, so our final objective is:
max⁡w,w01∣∣w∣∣min⁡n=1N[(w⊤x+w0)y~n]\max_{\bold{w},w_0} \frac{1}{||\bold{w}||}\min_{n=1}^N [(\bold{w}^\top \bold{x}+w_0)\tilde{y}_n]w,w0​max​∣∣w∣∣1​n=1minN​[(w⊤x+w0​)y~​n​]
Note that rescaling the parameters using w→kw\bold{w}\rightarrow k\bold{w}w→kw﻿ don’t change the distance to the decision boundary, since we divide by ∣∣w∣∣||\bold{w}||∣∣w∣∣﻿.
Therefore let’s define the scale factor such as fny~n=1f_n\tilde{y}_n=1fn​y~​n​=1﻿ that closest point to the decision boundary. Hence fny~n≥1f_n\tilde{y}_n\geq 1fn​y~​n​≥1﻿ for all nnn﻿.
Thus, we get the new objective:
min⁡w,w012∣∣w∣∣2s.t.(w⊤x+w0)y~n≥1,n=1:N\min_{\bold{w},w_0} \frac{1}{2}||\bold{w}||^2 \quad s.t.\quad (\bold{w^\top}\bold{x}+w_0)\tilde{y}_n \geq 1,n=1:Nw,w0​min​21​∣∣w∣∣2s.t.(w⊤x+w0​)y~​n​≥1,n=1:N
Note that it is important to scale the input variables before using an SVM, otherwise the margin measures distance of a point to the boundary using all input dimensions equally.

17.3.2 The dual problem

The last objective is a standard quadratic programming problem since we have a quadratic objective subject to linear constraints. This has N+D+1N+D+1 N+D+1﻿ variables for NNN﻿ constraints, and is known as a primal problem.
In convex optimization, for every primal problem we can derive a dual problem. Let α∈RN\alpha \in \R^Nα∈RN﻿ be the dual variables, corresponding to Lagrangian multipliers that enforce NNN﻿ inequality constraints.
The generalized Lagrangian is:
L(w,w0,α)=12w⊤w−∑n=1Nαn((w⊤xn+w0)y~n−1)\mathcal{L}(\bold{w},w_0,\alpha)=\frac{1}{2}\bold{w}^\top \bold{w}-\sum_{n=1}^N\alpha_n ((\bold{w}^\top \bold{x}_n+w_0)\tilde{y}_n-1)L(w,w0​,α)=21​w⊤w−n=1∑N​αn​((w⊤xn​+w0​)y~​n​−1)

(w^,w^0,α^)=arg min⁡w,w0max⁡αL(w,w0,α)(\hat{\bold{w}},\hat{w}_0,\hat{\alpha})=\argmin_{\bold{w},w_0} \max_\alpha\mathcal{L}(\bold{w},w_0,\alpha)(w^,w^0​,α^)=w,w0​argmin​αmax​L(w,w0​,α)

w^=∑n=1Nαny~nxn0=∑n=1Nαny~n\begin{align} \hat{\bold{w}} &=\sum_{n=1}^N \alpha_n \tilde{y}_n \bold{x}_n \\ 0&= \sum_{n=1}^N \alpha_n\tilde{y}_n \end{align}w^0​=n=1∑N​αn​y~​n​xn​=n=1∑N​αn​y~​n​​​

This is called the dual form of the objective. We want to maximize this wrt α\alphaα﻿ subject to the constraints that ∑n=1Nαny~n=0\sum_{n=1}^N\alpha_n \tilde{y}_n=0∑n=1N​αn​y~​n​=0﻿ and αn≥0\alpha_n\geq 0αn​≥0﻿​

17.3.3 Soft margin classifiers

If the data is not linearly separable, there will be no feasible solution in which y~nfn≥1\tilde{y}_nf_n\geq 1y~​n​fn​≥1﻿ for all nnn﻿.
We therefore introduce slack variables ξn≥0\xi_n\geq 0ξn​≥0﻿ and replace the hard constraints with soft margin constraints y~nfn≥1−ξn\tilde{y}_nf_n\geq 1-\xi_ny~​n​fn​≥1−ξn​﻿.
The objective becomes:
min⁡w,w012∣∣w∣∣2+C∑n=1Nξns.t.ξn≥0,(w⊤x+w0)y~n≥1−ξn\min_{\bold{w},w_0} \frac{1}{2}||\bold{w}||^2+C\sum_{n=1}^N\xi_n \quad s.t.\quad \xi_n\ge 0,\quad(\bold{w^\top}\bold{x}+w_0)\tilde{y}_n \geq 1-\xi_nw,w0​min​21​∣∣w∣∣2+Cn=1∑N​ξn​s.t.ξn​≥0,(w⊤x+w0​)y~​n​≥1−ξn​
where C≥0C\geq 0C≥0﻿ is a hyperparameter controlling how many points we allow to violate the margin constraint. If C=∞C=\infinC=∞﻿, we recover the previous unregularized hard margin classifier.
The corresponding Lagrangian is:
L(w,w0,α,ξ,μ)=12∣∣w∣∣2+C∑n=1Nξn+∑n=1Nαn((w⊤xn+w0)y~n−1+ξn)+∑n=1Nμnξn\begin{align} \mathcal{L}(\bold{w},w_0,\alpha,\xi,\mu)&=\frac{1}{2}||\bold{w}||^2+C\sum_{n=1}^N \xi_n \\&+\sum_{n=1}^N \alpha_n((\bold{w}^\top \bold{x}_n+w_0)\tilde{y}_n-1+\xi_n)+\sum_{n=1}^N\mu_n \xi_n \end{align}L(w,w0​,α,ξ,μ)​=21​∣∣w∣∣2+Cn=1∑N​ξn​+n=1∑N​αn​((w⊤xn​+w0​)y~​n​−1+ξn​)+n=1∑N​μn​ξn​​​
where αn\alpha_n αn​﻿ and μn\mu_nμn​﻿ are the Lagrange multipliers.
Optimizing out w\bold{w}w﻿, w0w_0w0​﻿ and ξ\xiξ﻿ gives the dual form:
L(α)=−12∣∣w∣∣2+∑n=1Nαn\mathcal{L}(\alpha)=-\frac{1}{2}||\bold{w}||^2+\sum_{n=1}^N \alpha_nL(α)=−21​∣∣w∣∣2+n=1∑N​αn​
This is identical to the hard margin case, however the constraints are different. The KKT conditions imply that:
0≤αn≤C∑n=1Nαny~n=0\begin{align} 0\leq \alpha_n &\leq C \\ \sum_{n=1}^N \alpha_n \tilde{y}_n &= 0 \end{align}0≤αn​n=1∑N​αn​y~​n​​≤C=0​​
If αn=0\alpha_n=0αn​=0﻿, the point is ignored.
If 0<αn<C0<\alpha_n<C0<αn​<C﻿ then ξn=0\xi_n=0ξn​=0﻿ so the point lies on the margin.
If αn=C\alpha_n=Cαn​=C﻿, then the point can lie inside the margin, and can either be correctly classified if ξn≤1\xi_n\leq 1ξn​≤1﻿, or misclassified if ξ>1\xi > 1ξ>1﻿.
Hence, ∑n=1Nξn\sum_{n=1}^N \xi_n∑n=1N​ξn​﻿ is an upper bound on the number of misclassified points.
As before, the bias term can be computed using:
w^0=1∣M∣∑n∈M(y~n−w⊤xn)\hat{w}_0=\frac{1}{|\mathcal{M}|}\sum_{n\in\mathcal{M}}(\tilde{y}_n-\bold{w}^\top\bold{x}_n)w^0​=∣M∣1​n∈M∑​(y~​n​−w⊤xn​)
where M\mathcal{M}M﻿ is the set of points having 0<αn<C0<\alpha_n<C0<αn​<C﻿.

subject to the constraints:

This has the advantage that the parameter ν\nuν﻿, which replace the parameter CCC﻿, can be interpreted as an upper bound on the fraction of margin errors (point for which ξn≥0\xi_n\ge 0ξn​≥0﻿), as well as a lower bound on the number of support vectors.

17.3.4 The kernel trick

We have converted the large margin classification problem into a dual problem in NN N﻿ unknowns αn\alpha_nαn​﻿ which takes O(N3)O(N^3)O(N3)﻿ time to solve, which is slow.
The benefit of the dual problem is that we can replace all inner product operation x⊤x′\bold{x}^\top \bold{x}'x⊤x′﻿ with a call to a positive definite Mercer kernel function, K(x,x′)\mathcal{K}(\bold{x},\bold{x}')K(x,x′)﻿. This is called kernel trick.
We can rewrite the prediction equation as:
f(x)=w^⊤x+w^0=∑n∈Sαny~nxn⊤x+w^0=∑n∈Sαny~nK(xn,x)+w^0\begin{align} f(\bold{x})=\hat{\bold{w}}^\top\bold{x}+\hat{w}_0&=\sum_{n\in\mathcal{S}}\alpha_n\tilde{y}_n \bold{x}_n^\top \bold{x}+\hat{w}_0\\ &= \sum_{n\in\mathcal{S}}\alpha_n\tilde{y}_n \mathcal{K}(\bold{x}_n,\bold{x})+\hat{w}_0\\ \end{align}f(x)=w^⊤x+w^0​​=n∈S∑​αn​y~​n​xn⊤​x+w^0​=n∈S∑​αn​y~​n​K(xn​,x)+w^0​​​
We also need to kernelize the bias term:
w^0=1∣S∣(∑n∈Sy~n−(∑m∈Sαmy~mxm⊤)xn)=1∣S∣(∑n∈Sy~n−∑m∈Sαmy~mK(xm,xn))\begin{align} \hat{w}_0&=\frac{1}{|\mathcal{S}|}\Big(\sum_{n\in\mathcal{S}} \tilde{y}_n-(\sum_{m\in\mathcal{S}}\alpha_m\tilde{y}_m\bold{x}_m^\top) \bold{x_n}\Big) \\ &= \frac{1}{|\mathcal{S}|}\Big(\sum_{n\in\mathcal{S}} \tilde{y}_n-\sum_{m\in\mathcal{S}}\alpha_m\tilde{y}_m\mathcal{K}(\bold{x}_m, \bold{x_n})\Big) \end{align}w^0​​=∣S∣1​(n∈S∑​y~​n​−(m∈S∑​αm​y~​m​xm⊤​)xn​)=∣S∣1​(n∈S∑​y~​n​−m∈S∑​αm​y~​m​K(xm​,xn​))​​
The kernel trick allows us to avoid dealing with explicit feature representation, and allows us to easily apply classifiers to structured objects, such as strings and graphs.

17.3.5 Converting SVM outputs into probabilities

An SVM classifier produce hard labeling y^(x)=sign(f(x))\hat{y}(\bold{x})=\mathrm{sign}(f(\bold{x}))y^​(x)=sign(f(x))﻿. However, we often want a measure of confidence in our prediction.
One heuristic approach is to interpret f(x)f(\bold{x})f(x)﻿ as the log-odds ratio log⁡p(y=1∣x)p(y=0∣x)\log \frac{p(y=1|\bold{x})}{p(y=0|\bold{x})}logp(y=0∣x)p(y=1∣x)​﻿. We can then convert an SVM output to probability:
p(y=1∣x,θ)=σ(af(x)+b)p(y=1|\bold{x},\theta)=\sigma(af(\bold{x})+b)p(y=1∣x,θ)=σ(af(x)+b)
where aaa﻿, bbb﻿ can be estimated using MLE on a separate validation set. This is known as Platt scaling.
However, the resulting probabilities are not well calibrated, since there is nothing that justifies interpreting f(x)f(\bold{x})f(x)﻿ as the log-odds ratio.
To see this, suppose we have 1d data where:
p(x∣y=0)=Uni(0,1)p(\bold{x}|y=0)=\mathrm{Uni}(0,1)p(x∣y=0)=Uni(0,1)﻿​
p(x∣y=1)=Uni(0.5,1.5)p(\bold{x}|y=1)=\mathrm{Uni}(0.5,1.5)p(x∣y=1)=Uni(0.5,1.5)﻿​
Since the class-conditional distributions overlap in this interval, the log-odd should be 0 inside it and infinite outside.
We train a probabilistic kernel classifier (RVM) and a SVM with Gaussian kernel. Both model can perfectly capture the decision boundary and achieve a generalization error of 25%, which is Bayes optimal in this problem.
However, the SVM doesn’t yield a good approximation of the true log-odds.

17.3.6 Connection with logistic regression

Data points that are on the (correct side) decision boundary have ξn=0\xi_n=0ξn​=0﻿, the others have ξn=1−y~nf(xn)\xi_n=1-\tilde{y}_nf(\bold{x}_n)ξn​=1−y~​n​f(xn​)﻿. Therefore we can rewrite the soft-margin objective in its primal form as:
L(w)=1N∑n=1Nℓhinge(y~n,f(xn))+λ∣∣w∣∣2\mathcal{L}(\bold{w})=\frac{1}{N}\sum_{n=1}^N \ell_{\mathrm{hinge}}(\tilde{y}_n,f(\bold{x}_n))+\lambda||\bold{w}||^2L(w)=N1​n=1∑N​ℓhinge​(y~​n​,f(xn​))+λ∣∣w∣∣2
where λ=12C\lambda=\frac{1}{2C}λ=2C1​﻿ and the hinge loss is defined as:
ℓhinge(y~,μ)=max⁡(0,1−y~μ)\ell_{\mathrm{hinge}}(\tilde{y},\mu)=\max(0, 1-\tilde{y}\mu)ℓhinge​(y~​,μ)=max(0,1−y~​μ)
This is a convex, piecewise differentiable upper-bound to the 0-1 loss.
By contrast, the (penalized) logistic regression optimizes:
L(w)=1N∑n=1Nℓll(y~n,f(xn))+λ∣∣w∣∣2\mathcal{L}(\bold{w})=\frac{1}{N}\sum_{n=1}^N \ell_{ll}(\tilde{y}_n,f(\bold{x}_n))+\lambda||\bold{w}||^2L(w)=N1​n=1∑N​ℓll​(y~​n​,f(xn​))+λ∣∣w∣∣2
where the log loss is given by:
ℓll(y~,μ)=−log⁡(p(y∣μ))=log⁡(1+e−y~μ)\ell_{ll}(\tilde{y},\mu)=-\log(p(y|\mu))=\log(1+e^{-\tilde{y}\mu})ℓll​(y~​,μ)=−log(p(y∣μ))=log(1+e−y~​μ)
The two major differences between both loss is that:
The hinge loss is piecewise linear, so we can’t use regular gradient methods to optimize it.
The hinge loss has a region where it is strictly 0. This results in sparse estimates.

17.3.7 Multi-class classification with SVMs

SVMs are inherently binary classifiers.
One way to convert them to a multi-class classification model is to train CCC﻿ binary classifiers, where the data from class ccc﻿ is treated as positive, and the rest is negative.
We then predict the label by using the rule:
y^(x)=arg max⁡cfc(x)\hat{y}(\bold{x})=\argmax_{c}f_c(\bold{x})y^​(x)=cargmax​fc​(x)
where fc(x)=log⁡p(c=1∣x)p(c=0∣x)f_c(\bold{x})=\log \frac{p(c=1|\bold{x})}{p(c=0|\bold{x})}fc​(x)=logp(c=0∣x)p(c=1∣x)​﻿. This is known as the one-vs-rest approach (aka one-vs-all).
Unfortunately, this has shortcomings:
This can result in regions of input space where the predicted label is ambiguous
The magnitude of the fcf_cfc​﻿ scores are not calibrated well with each other, so there are hard to compare.
Each binary subproblem is likely to suffer from imbalance problem, which can hurt performances: for 10 equally represented classes, each positive class represents only 10% of labels.
We can also use the one-vs-one (OVO) approach, in which we train C(C−1)/2C(C-1)/2C(C−1)/2﻿ classifiers to discriminate all pairs fc,c′f_{c,c'}fc,c′​﻿. We then classify a point into a class that has the higher number of votes.
However this can also result in ambiguities, and this require fitting O(C2)O(C^2) O(C2)﻿ models.

17.3.8 How to choose the regularizer CCC﻿​

SVMs require to specify the kernel function and the parameter CCC﻿, usually chosen with cross-validation.
Note however that CCC﻿ interact quite strongly with kernel parameters, for example with precision γ=1/2σ2\gamma=1/2\sigma^2γ=1/2σ2﻿.
If γ\gammaγ﻿ is large, corresponding to narrow kernel, we may need heavy regularization, hence small CCC﻿​
If γ\gamma γ﻿ is small, a larger value of CCC﻿ should be used.
The authors of libsvm recommend using CV over a 2d grid with values C∈{2−5,2−3,…,215}C\in\{2^{-5},2^{-3},\dots,2^{15}\}C∈{2−5,2−3,…,215}﻿ and γ∈{2−15,2−13,…,23}\gamma\in\{2^{-15},2^{-13},\dots,2^3\}γ∈{2−15,2−13,…,23}﻿​
To choose CCC﻿ efficiently, we can develop a path following algorithm similar to lars.
The idea is to start with a small CCC﻿ so that the margin is wide and all points are inside it. By increasing CCC﻿, points will move from inside the margin to outside, and their αi\alpha_iαi​﻿ value change from 1 to 0 as they cease to be support vectors. When CCC﻿ is maximal, there are no support vectors left.

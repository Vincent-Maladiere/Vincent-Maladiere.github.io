18.5 Boosting

Ensemble of trees, whether fit using bagging or the random forest algorithm, corresponds to a model of the form:
f(x;θ)=∑m=1MβmFm(x;θm)f(\bold{x};\theta)=\sum_{m=1}^M \beta_m F_m(\bold{x};\theta_m)f(x;θ)=m=1∑M​βm​Fm​(x;θm​)
where FmF_mFm​﻿ is the mmm﻿th tree and βm\beta_mβm​﻿ is the corresponding weight, often set to βm=1/M\beta_m=1/Mβm​=1/M﻿.
We can generalize this by allowing the FmF_mFm​﻿ functions to be general function approximators, such as neural nets. The result is called additive model.
We can think of this as a linear function with adaptative basis functions. The goal, as usual, is to minimize the empirical loss function.
The original boosting algorithm sequentially fits additive models where each FmF_mFm​﻿ is a binary classifiers that returns Fm∈{−1,+1}F_m\in\{-1,+1\}Fm​∈{−1,+1}﻿.
We first fit F1F_1F1​﻿ on the original dataset, then we weight the data samples by the errors made by F1F_1F1​﻿, so misclassified example get more weight.
Next, we fit F2F_2F2​﻿ on this weighted dataset.
We keep repeating this process until MMM﻿ number of components have been fitted (where MMM﻿ is a hyper-parameter that controls the complexity of the overall-model, and can be chosen by monitoring the performance on a validation dataset, via early-stopping).
It can be shown that as long as each FmF_mFm​﻿ has an accuracy above chance level (even slightly above 50%), then the final ensemble of classifiers will have a higher accuracy that any individual FmF_mFm​﻿.
In other words, we can boost a weak learner FmF_mFm​﻿ into a strong learner.
Note that boosting reduces the bias of strong learner by fitting trees that depends on each other, whereas bagging and RF reduces the variance by fitting independent trees.
In the rest of this section, we focus on boosting model with arbitrary loss function, suitable for regression, multiclass classification or ranking.

18.5.4 LogitBoost

The trouble with exponential loss is that it puts a lot of weight on misclassified examples.
In addition, e−yfe^{-yf}e−yf﻿ is not the logarithm of any pmf for binary variables y~∈{−1,+1}\tilde{y}\in\{-1,+1\}y~​∈{−1,+1}﻿, hence we can’t recover probabilities from f(x)f(\bold{x})f(x)﻿.
A natural alternative is to use log-loss, which only punishes mistakes linearly, and we’ll be able to extract probability using:
p(y=1∣x)=ef(x)ef(x)+e−f(x)=11+e−2f(x)p(y=1|\bold{x})=\frac{e^{f(\bold{x})}}{e^{f(\bold{x})}+e^{-f(\bold{x})}}=\frac{1}{1+e^{-2f(\bold{x})}}p(y=1∣x)=ef(x)+e−f(x)ef(x)​=1+e−2f(x)1​

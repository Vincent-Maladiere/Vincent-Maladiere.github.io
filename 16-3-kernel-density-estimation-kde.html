16.3.1 Density kernels

Density kernels are function K:R→R+\mathcal{K}:\R \rightarrow \R_+K:R→R+​﻿, such that ∫K(x)dx=1\int \mathcal{K}(x)dx=1∫K(x)dx=1﻿ and K(x)=K(−x)\mathcal{K}(x)=\mathcal{K}(-x)K(x)=K(−x)﻿.
This latter symmetry property implies ∫xK(x)dx=0\int x\mathcal{K}(x)dx=0∫xK(x)dx=0﻿ and hence:
∫xK(x−xn)dx=xn\int x\mathcal{K}(x-x_n)dx = x_n∫xK(x−xn​)dx=xn​
A simple example of such a kernel is the boxcar kernel, which is the uniform distribution around within the unit interval around the origin:
K(x)≜0.5I(∣x∣≤1)\mathcal{K}(x)\triangleq 0.5\mathbb{I}(|x|\le 1)K(x)≜0.5I(∣x∣≤1)
Another example is the Gaussian kernel:
K(x)≜12πe−x2/2\mathcal{K}(x)\triangleq \frac{1}{\sqrt{2\pi}}e^{-x^2/2}K(x)≜2π​1​e−x2/2
We can control the width of the kernel by introducing a bandwidth parameter hhh﻿:
Kh(x)≜xhK(xh)\mathcal{K}_h(x)\triangleq \frac{x}{h}\mathcal{K}(\frac{x}{h})Kh​(x)≜hx​K(hx​)
We can generalize to vector valued inputs by defining a radial basis function (RBF) kernel:
Kh(x)∝Kh(∣∣x∣∣)\mathcal{K}_h(\bold{x})\propto \mathcal{K}_h(||\bold{x}||)Kh​(x)∝Kh​(∣∣x∣∣)
In the case of the Gaussian kernel this becomes:
Kh(x)=1(2π)D/2∏d=1Dexp⁡(−xd22h2)\mathcal{K}_h(\bold{x})=\frac{1}{(2\pi)^{D/2}}\prod_{d=1}^D \exp({-\frac{x_d^2}{2h^2}})Kh​(x)=(2π)D/21​d=1∏D​exp(−2h2xd2​​)
Although the Gaussian kernel is popular, it has unbounded support. Compact kernels can be faster to compute.

16.3.2 Parzen window density estimator

We now explain how to use kernels to define a nonparametric density estimate.
Recall the form of Gaussian mixture, with a fixed spherical Gaussian covariance and uniform mixture weights:
p(x∣θ)=1K∑k=1Kp(x∣μk,σ2I)p(\bold{x}|\theta)=\frac{1}{K}\sum_{k=1}^K p(\bold{x}|\mu_k,\sigma^2 I)p(x∣θ)=K1​k=1∑K​p(x∣μk​,σ2I)
One problem with this model is that it requires specifying the number KKK﻿ of clusters and their positions μk\mu_kμk​﻿.
An alternative is to allocate one cluster center per data point:
p(x∣θ)=1N∑n=1Np(x∣xn,σ2I)p(\bold{x}|\theta)=\frac{1}{N}\sum_{n=1}^N p(\bold{x}|\bold{x}_n,\sigma^2I)p(x∣θ)=N1​n=1∑N​p(x∣xn​,σ2I)
This can be generalized to:
p(x∣θ)=1N∑n=1NKh(x−xn)p(\bold{x}|\theta)=\frac{1}{N}\sum_{n=1}^N \mathcal{K}_h(\bold{x}-\bold{x}_n)p(x∣θ)=N1​n=1∑N​Kh​(x−xn​)
This is called a Parzen window density estimator or kernel density estimator (KDE).
Its advantage over a parametric model is that no fitting is required (except for choosing hhh﻿) and there is no need to select the clusters.
The drawback is that it takes a lot of memory and a lot of time to evaluate.
The resulting model using the boxcar kernel simply count the number of points within a window of size hhh﻿. The Gaussian kernel gives a smoother density.

16.3.3 How to choose the bandwith parameter

The bandwidth parameter hhh﻿ controls the complexity of the model.
In 1d data, if we assume the data has been generated from a Gaussian distribution, one can show the bandwidth minimizing the frequentist risk is given by:
h=σ(43N)1/5h=\sigma(\frac{4}{3N})^{1/5}h=σ(3N4​)1/5

and then using σ^=1.4826 MAD\hat{\sigma}=1.4826 \,\mathrm{MAD}σ^=1.4826MAD﻿.

16.3.4 From KDE to KNN classification

We previously discussed the KKK﻿ neighbor classifier as a heuristic approach to classification. Interestingly, we can derive it as a generative classifier in which the class conditional densities p(x∣y=c)p(\bold{x}|y=c)p(x∣y=c)﻿ are modeled using a KDE.
Rather than a fixed bandwidth a counting the points within a hypercube centered on a datapoint, we allow the bandwidth to be different for each point.
We “grow” a volume around x\bold{x}x﻿ until we captured KKK﻿ points, regardless of their class label. This is called a balloon density kernel estimator.
Let the resulting volume have size V(x)V(\bold{x})V(x)﻿ (this was previously hD)h^D)hD)﻿ and let there Nc(x)N_c(\bold{x})Nc​(x)﻿ example of the class ccc﻿ in this volume. We can then estimate the class conditional density:
p(x∣y=c)=Nc(x)V(x)Ncp(\bold{x}|y=c)=\frac{N_c(\bold{x})}{V(\bold{x})N_c}p(x∣y=c)=V(x)Nc​Nc​(x)​
where NcN_cNc​﻿ is the total number of point with class labels ccc﻿ in the dataset.
If we take the class prior to be Nc/NN_c/NNc​/N﻿, we have the posterior:
p(y=c∣x,D)=Nc(x)V(x)NcNcN∑c′Nc′(x)V(x)Nc′Nc′N=Nc(x)∑c′Nc′=Nc(x)Kp(y=c|\bold{x},\mathcal{D})=\frac{\frac{N_c(\bold{x})}{V(\bold{x})N_c}\frac{N_c}{N}}{\sum_{c'}\frac{N_{c'}(\bold{x})}{V(\bold{x})N_{c'}}\frac{N_{c'}}{N}}=\frac{N_c(\bold{x})}{\sum_{c'} N_{c'}}=\frac{N_c(\bold{x})}{K}p(y=c∣x,D)=∑c′​V(x)Nc′​Nc′​(x)​NNc′​​V(x)Nc​Nc​(x)​NNc​​​=∑c′​Nc′​Nc​(x)​=KNc​(x)​

16.3.5 Kernel regression

Just as KDE can be use for generative classifiers, it can also be used as generative models for regression.
16.3.5.1 Nadaraya-Watson estimator for the mean
In regression, our goal is to compute:
E[y∣x,D]=∫yp(y∣x,D)dy=∫yp(x,y∣D)dy∫p(x,y∣D)dy\mathbb{E}[y|x,\mathcal{D}]=\int y p(y|\bold{x},\mathcal{D})dy=\frac{\int yp(\bold{x},y|\mathcal{D})dy}{\int p(\bold{x},y|\mathcal{D})dy}E[y∣x,D]=∫yp(y∣x,D)dy=∫p(x,y∣D)dy∫yp(x,y∣D)dy​
If we use a MVN for p(x,y∣D)p(\bold{x},y|\mathcal{D})p(x,y∣D)﻿, we derive a result which is equivalent to linear regression.
However, the Gaussian assumption on p(x,y∣D)p(\bold{x},y|\mathcal{D})p(x,y∣D)﻿ is rather limiting. We can use KDE to more accurately approximate this joint density:
p(x,y∣D)≈1N∑n=1NKh(x−xn)Kh(y−yn)p(\bold{x},y|\mathcal{D})\approx \frac{1}{N}\sum_{n=1}^N\mathcal{K}_h(\bold{x}-\bold{x}_n)\mathcal{K}_h(y-y_n)p(x,y∣D)≈N1​n=1∑N​Kh​(x−xn​)Kh​(y−yn​)
Hence, using the previous kernel properties:
E[y∣x,D]=∑n=1NKh(x−xn)∫yKh(y−yn)dy∑n=1NKh(x−xn)∫Kh(y−yn)dy=∑n=1Nynwn(x)\begin{align} \mathbb{E}[y|\bold{x},\mathcal{D}] &=\frac{\sum_{n=1}^N\mathcal{K}_h(\bold{x}-\bold{x}_n)\int y\mathcal{K}_h(y-y_n)dy}{\sum_{n=1}^N\mathcal{K}_h(\bold{x}-\bold{x}_n)\int \mathcal{K}_h(y-y_n)dy} \\ &= \sum_{n=1}^N y_nw_n(\bold{x}) \end{align}E[y∣x,D]​=∑n=1N​Kh​(x−xn​)∫Kh​(y−yn​)dy∑n=1N​Kh​(x−xn​)∫yKh​(y−yn​)dy​=n=1∑N​yn​wn​(x)​​
where:
wn(x)=Kh(x−xn)∑n′=1NKh(x−xn′)w_n(\bold{x})=\frac{\mathcal{K}_h(\bold{x}-\bold{x}_n)}{\sum_{{n'}=1}^N \mathcal{K}_h(\bold{x}-\bold{x}_{n'})}wn​(x)=∑n′=1N​Kh​(x−xn′​)Kh​(x−xn​)​
We see that the prediction is just a weighted sum of the training labels, where the weights depend on the similarity between x\bold{x}x﻿ and the stored training points.
This method is called kernel regression, kernel smoothing, or the Nadaraya-Watson (N-W) model.
16.3.5.2 Estimator for the variance
Sometimes it can be useful to compute the predictive variance, as well as the predictive mean. We can do this by noting that:
V[y∣x,D]=E[y2∣x,D]−μ(x)2\mathbb{V}[y|\bold{x},\mathcal{D}]=\mathbb{E}[y^2|\bold{x},\mathcal{D}]-\mu(\bold{x})^2V[y∣x,D]=E[y2∣x,D]−μ(x)2
where μ(x)=E[y∣x,D]\mu(\bold{x})=\mathbb{E}[y|\bold{x},\mathcal{D}]μ(x)=E[y∣x,D]﻿ is the N-W estimate.
If we use a Gaussian kernel with variance σ2\sigma^2σ2﻿, we can compute:
E[y2∣x,D]=∑n=1NKh(x−xn)∫y2Kh(y−yn)dy∑n=1NKh(x−xn)∫Kh(y−yn)dy=∑n=1NKh(x−xn)(σ2+yn2)∑n=1NKh(x−xn)\begin{align} \mathbb{E}[y^2|\bold{x},\mathcal{D}] &=\frac{\sum_{n=1}^N \mathcal{K}_h(\bold{x}-\bold{x}_n)\int y^2\mathcal{K}_h(y-y_n)dy}{\sum_{n=1}^N \mathcal{K}_h(\bold{x}-\bold{x}_n)\int \mathcal{K}_h(y-y_n)dy}\\ &= \frac{\sum_{n=1}^N \mathcal{K}_h(\bold{x}-\bold{x}_n)(\sigma^2+y^2_n)}{\sum_{n=1}^N \mathcal{K}_h(\bold{x}-\bold{x}_n)} \end{align}E[y2∣x,D]​=∑n=1N​Kh​(x−xn​)∫Kh​(y−yn​)dy∑n=1N​Kh​(x−xn​)∫y2Kh​(y−yn​)dy​=∑n=1N​Kh​(x−xn​)∑n=1N​Kh​(x−xn​)(σ2+yn2​)​​​
where we used the fact that:
∫y2Kh(y−yn)dy=σ2+yn2\int y^2\mathcal{K}_h(y-y_n)dy=\sigma^2+y_n^2∫y2Kh​(y−yn​)dy=σ2+yn2​
Finally:
V[y∣x,D]=σ2+∑n=1Nyn2wn(x)−μ(x)2\mathcal{V}[y|\bold{x},\mathcal{D}]=\sigma^2+\sum_{n=1}^Ny_n^2w_n(\bold{x})-\mu(\bold{x})^2V[y∣x,D]=σ2+n=1∑N​yn2​wn​(x)−μ(x)2
16.3.5.3 Locally weighted regression
We can drop the normalization term to get:
μ(x)=∑n=1NynK(x−xn)\mu(\bold{x})=\sum_{n=1}^N y_n \mathcal{K}(\bold{x}-\bold{x}_n)μ(x)=n=1∑N​yn​K(x−xn​)
Rather than interpolating the stored labels yny_nyn​﻿, we can fit a locally linear model around each training point:
μ(x)=min⁡β∑n=1N(yn−β⊤xn)2Kh(x−xn)\mu(\bold{x})=\min_{\beta}\sum_{n=1}^N (y_n-\beta^\top \bold{x}_n)^2\mathcal{K}_h(\bold{x}-\bold{x}_n)μ(x)=βmin​n=1∑N​(yn​−β⊤xn​)2Kh​(x−xn​)
This is called locally linear regression (LLR), or locally-weighted scatterplot smoothing (LOWESS or LOESS).
This is often used when annotating scatter plots with local trend lines.

11.3 Ridge regression

Maximum likelihood estimation can result in overfitting, a simple solution is to use MAP estimation with zero-mean Gaussian prior:
p(w)=N(w∣0,λ−1I)p(\bold{w})=\mathcal{N}(\bold{w}|0, \lambda^{-1} \mathrm{I})p(w)=N(w∣0,λ−1I)
We compute the MAP as:
wmap=arg min⁡wRSS(w)+λ∣∣w∣∣22\bold{w}_{map}=\argmin_{\bold{w}}\mathrm{RSS}(\bold{w}) + \lambda ||\bold{w}||_2^2wmap​=wargmin​RSS(w)+λ∣∣w∣∣22​
Therefore we are penalizing the weights that become too large in magnitude. This is called ℓ2\ell_2ℓ2​﻿ regularization or weight decay.
We don’t penalize the term w0w_0w0​﻿ since it doesn’t contribute to overfitting.

11.3.1 Computing the MAP estimate

The MAP estimation corresponds to computing the minimizing the objective:
J(w)=∣∣y−Xw∣∣22+λ∣∣w∣∣22J(\bold{w})=||\bold{y}-X\bold{w}||^2_2+ \lambda ||\bold{w}||^2_2J(w)=∣∣y−Xw∣∣22​+λ∣∣w∣∣22​
we have:
∇wJ(w)=2(X⊤Xw−X⊤y+λw)\nabla_\bold{w}J(\bold{w})=2(X^\top X\bold{w}-X ^\top \bold{y}+\lambda \bold{w})∇w​J(w)=2(X⊤Xw−X⊤y+λw)
Hence:
w^map=(X⊤X+λID)−1X⊤y=(∑nxnxn⊤+λID)−1(∑nxnyn)\begin{align} \hat{\bold{w}}_{map}&=(X^\top X+\lambda \mathrm{I}_D)^{-1}X^\top \bold{y} \\ &= (\sum_n \bold{x}_n\bold{x}_n^\top+\lambda \mathrm{I}_D)^{-1}(\sum_{n}\bold{x}_ny_n) \end{align}w^map​​=(X⊤X+λID​)−1X⊤y=(n∑​xn​xn⊤​+λID​)−1(n∑​xn​yn​)​​
11.3.1.1 Solving using QR
Naively performing the matrix inversion can be slow and numerically unstable. We propose a way to convert the problem to a standard least square, where we can apply QR decomposition as previously seen.
We assume the prior has the form N(0,Λ−1)\mathcal{N}(0,\Lambda^{-1})N(0,Λ−1)﻿ where Λ=(λ/σ2)I\Lambda =(\lambda/\sigma^2)\mathrm{I}Λ=(λ/σ2)I﻿ is the precision matrix.
We can emulate this prior by augmenting our training data as:
X~=[X/σΛ],y~=[y/σ0D]\tilde{X}=\begin{bmatrix}X/\sigma\\ \sqrt{\Lambda} \end{bmatrix},\quad \tilde{\bold{y}}=\begin{bmatrix}\bold{y/\sigma}\\\bold{0_{D}}\end{bmatrix}X~=[X/σΛ​​],y~​=[y/σ0D​​]
where Λ=ΛΛ⊤\Lambda=\sqrt{\Lambda}\sqrt{\Lambda}^\top Λ=Λ​Λ​⊤﻿ is the Cholesky decomposition.
We now show that the RSS of this expended data is equivalent to the penalized RSS on the original data:
f(w)=(y~−X~w)⊤(y~−X~w)=([y/σ0]−[X/σΛ]w)⊤([y/σ0]−[X/σΛ]w)=[1/σ(y−Xw)−Λw]⊤[1/σ(y−Xw)−Λw]=1σ2(y−Xw)⊤(y−Xw)+w⊤Λw\begin{align} f(\bold{w})&=(\bold{\tilde{y}}-\tilde{X}\bold{w})^\top(\tilde{\bold{y}}-\tilde{X}\bold{w}) \\ &= \Bigg(\begin{bmatrix}\bold{y}/\sigma\\\bold{0}\end{bmatrix} - \begin{bmatrix}X/\sigma\\\sqrt{\Lambda}\end{bmatrix}\bold{w}\Bigg)^\top \Bigg(\begin{bmatrix}\bold{y/\sigma}\\\bold{0}\end{bmatrix} - \begin{bmatrix}X/\sigma\\\sqrt{\Lambda}\end{bmatrix}\bold{w}\Bigg) \\ &=\begin{bmatrix}1/\sigma(\bold{y}-X\bold{w})\\-\sqrt{\Lambda}\bold{w}\end{bmatrix}^\top \begin{bmatrix}1/\sigma(\bold{y}-X\bold{w})\\-\sqrt{\Lambda }\bold{w}\end{bmatrix}\\ &= \frac{1}{\sigma^2}(\bold{y}-X\bold{w})^\top(\bold{y}-X\bold{w})+\bold{w}^\top\Lambda\bold{w} \end{align}f(w)​=(y~​−X~w)⊤(y~​−X~w)=([y/σ0​]−[X/σΛ​​]w)⊤([y/σ0​]−[X/σΛ​​]w)=[1/σ(y−Xw)−Λ​w​]⊤[1/σ(y−Xw)−Λ​w​]=σ21​(y−Xw)⊤(y−Xw)+w⊤Λw​​
Hence, the MAP estimate is given by:
w^map=(X~⊤X~)−1X~⊤y~\hat{\bold{w}}_{map}=(\tilde{X}^\top \tilde{X})^{-1}\tilde{X}^\top \bold{\tilde{y}}w^map​=(X~⊤X~)−1X~⊤y~​
And then solving using standard OLS method, by computing the QR decomposition of X~\tilde{X}X~﻿. This takes O((N+D)D2)O((N+D)D^2)O((N+D)D2)﻿​
11.3.1.2 Solving using SVD
In this section, we assume D>ND>ND>N﻿, which is a framework that suits ridge regression well. In this case, it is faster to compute SVD than QR.
Let X=USV=RVX=USV=RVX=USV=RV﻿, with
U∈RN×NU\in\mathbb{R}^{N\times N}U∈RN×N﻿, so that UU⊤=U⊤U=INUU^\top =U^\top U=\mathrm{I}_NUU⊤=U⊤U=IN​﻿
V∈RN×DV\in\mathbb{R}^{N\times D}V∈RN×D﻿, so that V⊤V=INV^\top V =I_NV⊤V=IN​﻿​
R∈RN×NR\in\mathbb{R}^{N\times N}R∈RN×N﻿
One can show that:
w^map=V(R⊤R+λIN)−1R⊤y\hat{\bold{w}}_{map}= V(R^\top R+\lambda \mathrm{I}_N)^{-1}R^\top \bold{y}w^map​=V(R⊤R+λIN​)−1R⊤y
In other words, we can replace the DDD﻿-dimensional vector xi\bold{x}_ixi​﻿ with NNN﻿-dimensional vector ri\bold{r}_iri​﻿ and perform our penalized fit as before.
The resulting complexity is O(DN2)O(DN^2)O(DN2)﻿, which is less than O(D3)O(D^3)O(D3)﻿ if N<DN<DN<D﻿​

11.3.2 Connection between ridge regression and PCA

The ridge predictions on the training set are given by:
y^=Xw^=USV⊤V(S2+λIN)−1SU⊤y=US~U⊤y=∑j=1DujS~jjuj⊤y\begin{align} \hat{\bold{y}}=X\bold{\hat{w}}&=USV^\top V(S^2+\lambda I_N)^{-1}SU^\top \bold{y}\\ &= U\tilde{S}U^\top \bold{y}=\sum_{j=1}^D \bold{u}_j \tilde{S}_{jj}\bold{u}_j^\top \bold{y} \end{align}y^​=Xw^​=USV⊤V(S2+λIN​)−1SU⊤y=US~U⊤y=j=1∑D​uj​S~jj​uj⊤​y​​
with:
S~jj≜[S(S2+λIN)−1S]jj=σj2λ+σj2\tilde{S}_{jj}\triangleq[S(S^2+\lambda I_N)^{-1}S]_{jj}=\frac{\sigma_j^2}{\lambda +\sigma_j^2}S~jj​≜[S(S2+λIN​)−1S]jj​=λ+σj2​σj2​​
Hence:
y^=Xw^map=∑j=1Dujσj2σj2+λuj⊤y\hat{\bold{y}}=X\hat{\bold{w}}_{map}=\sum_{j=1}^D \bold{u}_j\frac{\sigma^2_j}{\sigma^2_j+\lambda}\bold{u}_j^\top \bold{y}y^​=Xw^map​=j=1∑D​uj​σj2​+λσj2​​uj⊤​y
In contrast, the least square prediction are:
y^=Xw^mle=∑j=1Dujuj⊤y\hat{\bold{y}}=X\hat{\bold{w}}_{mle}=\sum_{j=1}^D \bold{u}_j\bold{u}_j^\top \bold{y}y^​=Xw^mle​=j=1∑D​uj​uj⊤​y
If σj2≪λ\sigma^2_j\ll \lambdaσj2​≪λ﻿, the direction uj\bold{u}_juj​﻿ will have a small impact on the prediction. This is what we want, since small singular values corresponds to direction with high posterior variance. These are the directions ridge shrinks the most.
There is a related technique called principal components regression, a supervised PCA reducing dimensionality to KKK﻿ followed by a regression. However, this is usually less accurate than ridge, since it only uses KKK﻿ features, when ridge uses a soft-weighting of all the dimensions.

11.3.3 Choosing the strength of the regularizer

To find the optimal λ\lambdaλ﻿, we can run cross-validation on a finite set of values and get the expected loss.
This approach can be expensive for large set of hyper-parameters, but fortunately we can often warm-start the optimization procedure, using the value of w^(λk)\hat{\bold{w}}(\lambda_k)w^(λk​)﻿ as a initializer for w^(λk+1)\hat{\bold{w}}(\lambda_{k+1})w^(λk+1​)﻿.
If we set λk+1>λk\lambda_{k+1}>\lambda_kλk+1​>λk​﻿, we start from a high amount of regularization and gradually diminish it.
We can also use empirical Bayes to choose λ\lambdaλ﻿, by computing:
λ^=arg max⁡λlog⁡p(D∣λ)\hat{\lambda}=\argmax_\lambda \log p(\mathcal{D}|\lambda)λ^=λargmax​logp(D∣λ)
where p(D∣λ)p(\mathcal{D|\lambda})p(D∣λ)﻿ is the marginal likelihood.
This gives the same result as CV estimate, however the Bayesian approach only fit a single model, and p(D∣λ)p(\mathcal{D}|\lambda)p(D∣λ)﻿ is a smooth function of λ\lambdaλ﻿, so we can use gradient-based optimization instead of a discrete search.

14.6 Generating images by inverting CNNs

A CNN trained for image classification is a discriminative model of the form p(y∣x)p(y|\bold{x})p(y∣x)﻿, returning a probability distribution over CCC﻿ class labels.
In this section, we convert this model into a conditional generative image model of the form p(x∣y)p(\bold{x}|y)p(x∣y)﻿. This will allow us to generate images belonging to a specific class.

14.6.1 Converting a trained classifier into a generative model

We can define a joint distribution over images and labels p(x,y)=p(y∣x)p(x)p(\bold{x},y)=p(y|\bold{x})p(\bold{x})p(x,y)=p(y∣x)p(x)﻿.
If we select a specific label value, we can create a conditional generative model p(x∣y)∝p(y∣x)p(x)p(\bold{x}|y)\propto p(y|\bold{x})p(\bold{x})p(x∣y)∝p(y∣x)p(x)﻿.
Since p(y∣x)p(y|\bold{x})p(y∣x)﻿ is not an invertible function, the prior p(x)p(\bold{x})p(x)﻿ will play an important role as a regularizer.
One way to sample from this model is to use Metropolis Hasting algorithm, were the energy function is:
Ec(x)=log⁡p(y=c∣x)+log⁡(x)\mathcal{E}_c(\bold{x})=\log p(y=c|\bold{x})+\log (\bold{x})Ec​(x)=logp(y=c∣x)+log(x)
Since the gradient is available, we then make the update:
q(x′∣x)=N(x′∣μ(x),ϵI)q(\bold{x}'|\bold{x})=\mathcal{N}(\bold{x}'|\mu(\bold{x}),\epsilon I)q(x′∣x)=N(x′∣μ(x),ϵI)
where:
μ(x)=x+ϵ2∇log⁡Ec(x)\mu(\bold{x})=\bold{x}+\frac{\epsilon}{2}\nabla \log \mathcal{E}_c(\bold{x})μ(x)=x+2ϵ​∇logEc​(x)
This is called the Metropolis adjusted Langevin algorithm (MALA).
As an approximation we skip the rejection part and accept every candidate, this is called the unadjusted Lanvin algorithm and we used for conditional image generation.
Thus, we get an update over the space of images that looks like a noisy SGD, but the gradient are taken w.r.t the inputs instead of the weights:
xt+1=xt+ϵ1∂log⁡p(xt)∂xt+ϵ2∂log⁡p(y=c∣xt)∂xt+N(0,ϵ32I)\bold{x}_{t+1}=\bold{x}_t+\epsilon_1\frac{\partial \log p(\bold{x}_t)}{\partial \bold{x}_t}+\epsilon_2\frac{\partial \log p(y=c|\bold{x}_t)}{\partial \bold{x}_t}+\mathcal{N}(\bold{0},\epsilon_3^2I)xt+1​=xt​+ϵ1​∂xt​∂logp(xt​)​+ϵ2​∂xt​∂logp(y=c∣xt​)​+N(0,ϵ32​I)
ϵ1\epsilon_1ϵ1​﻿ ensures the image is plausible under the prior
ϵ2\epsilon_2ϵ2​﻿ ensures the image is plausible under the likelihood
ϵ3\epsilon_3ϵ3​﻿ is a noise term to enforce diversity. If set to 0, the method a deterministic algorithm generating the “most likely image” for this class.

14.6.2.2 Total variation (TV) prior
To help generating more realistic images, we can add extra regularizers like total variation or TV norm of an image. This is the integral of the per-pixel gradients, approximated as:
TV(x)=∑ijk(xi+1,j,k−xi,j,k)2+(xi,j+1,k−xi,j,k)2\mathrm{TV}(\bold{x})=\sum_{ijk}(x_{i+1,j,k}-x_{i,j,k})^2+(x_{i,j+1,k}-x_{i,j,k})^2TV(x)=ijk∑​(xi+1,j,k​−xi,j,k​)2+(xi,j+1,k​−xi,j,k​)2
where xi,j,kx_{i,j,k}xi,j,k​﻿ is the pixel value of at location (i,j)(i,j)(i,j)﻿ for channel kkk﻿.
We can rewrite this in term of the horizontal and vertical Sobel edge detector applied to each channel:
TV(x)=∑k∣∣H(x:,:,k)∣∣F2+∣∣V(x:,:,k∣∣F2)\mathrm{TV}(\bold{x})=\sum_k ||\bold{H}(\bold{x}_{:,:,k})||_F^2+||\bold{V}(\bold{x}_{:,:,k}||^2_F)TV(x)=k∑​∣∣H(x:,:,k​)∣∣F2​+∣∣V(x:,:,k​∣∣F2​)
Using:
p(x)∝e−TV(x)p(\bold{x})\propto e^{-\mathrm{TV}(\bold{x})}p(x)∝e−TV(x)
discourage images from having high frequency artifacts.
Gaussian blur can also be used instead of TV, with similar effects.

14.6.3 Visualizing the feature learned by a CNN

Activation maximization (AM) optimizes a random image to maximize the activation of a given layer. This can be useful to get a better grasp of the features learned by the ConvNet.

14.6.4 Deep Dream

Instead of generating images that maximize the feature map of a given layer or class labels, we now want to express these features over an input image.
We view or trained classifier as a feature extractor. To amplify features from layers l∈Ll\in\mathcal{L}l∈L﻿, we define a loss function of the form:
L(x)=∑l∈Lϕlˉ(x)\mathcal{L}(\bold{x})=\sum_{l\in\mathcal{L}}\bar{\phi_l}(\bold{x})L(x)=l∈L∑​ϕl​ˉ​(x)
where
ϕl(x)=1HWC∑hwcϕlhwc(x)\phi_l(\bold{x})=\frac{1}{HWC}\sum_{hwc}\phi_{lhwc}(\bold{x})ϕl​(x)=HWC1​hwc∑​ϕlhwc​(x)
is the feature vector for layer lll﻿.
We can now use gradient descent to optimise this loss and update our original image. By using the output as input in a loop, we iteratively add features to it.
This is called DeepDream, because the model amplifies features that were only initially hinted.
The output image above is a hybrid between the original image and “hallucinations” of dog parts, because ImageNet contains contains so many kinds of dogs.

14.6.5 Neural style transfer

Neural style transfer give more generative control to the user, by specifying a “content” image xc\bold{x}_cxc​﻿ and a “style” image xs\bold{x}_sxs​﻿. The model will try to generate new image x\bold{x}x﻿ that re-renders xc\bold{x}_cxc​﻿ by applying the style xs\bold{x}_sxs​﻿.
14.6.5.1 How it works
Style transfer works by optimizing the loss function:
L(x∣xs,xc)=λTVTV(x)+λsLs(x,xs)+λcLc(x,xc)\mathcal{L}(\bold{x}|\bold{x}_s,\bold{x}_c)=\lambda_{TV} \mathrm{TV}(\bold{x})+\lambda_s \mathcal{L}_s(\bold{x},\bold{x}_s)+\lambda_c \mathcal{L}_c(\bold{x},\bold{x}_c)L(x∣xs​,xc​)=λTV​TV(x)+λs​Ls​(x,xs​)+λc​Lc​(x,xc​)
where TV\mathrm{TV}TV﻿ is the total variation prior.
The content loss measures similarity between x\bold{x}x﻿ to xc\bold{x}_cxc​﻿ by comparing feature maps of a pre-trained ConvNet Φ\PhiΦ﻿ in the layer ℓ\ellℓ﻿:
Lc(x,xc)=1CℓHℓWℓ∣∣ϕℓ(x)−ϕℓ(xc)∣∣22\mathcal{L}_c(\bold{x},\bold{x}_c)=\frac{1}{C_\ell H_\ell W_\ell}||\phi_\ell(\bold{x})-\phi_\ell(\bold{x}_c)||_2^2Lc​(x,xc​)=Cℓ​Hℓ​Wℓ​1​∣∣ϕℓ​(x)−ϕℓ​(xc​)∣∣22​
We can interpret the style as a statistical distribution of some kinds of image features. Their location don’t matter, but their co-occurence does.
To capture the co-occurence statistics, we compute the Gram matrix for the image using feature maps at layer ℓ\ellℓ﻿:

14.6.5.2 Speeding up the method
In the Neural style transfer paper, they used L-BFGS to optimize the total loss, starting from white noise.
We can get faster results if we use Adam and we start from an actual image instead of noise. Nevertheless, running optimization for every single content and style image is slow.
i) Amortized optimization
Instead, we can train a network to predict the outcome of this optimization. This can be viewed as a form of amortized optimization.
We fit a model for every style image:
fs(xc)=arg max⁡xL(x∣xs,xc)f_s(\bold{x}_c)=\argmax_{\bold{x}}\mathcal{L}(\bold{x}|\bold{x}_s,\bold{x}_c)fs​(xc​)=xargmax​L(x∣xs​,xc​)
We can apply this to new content images without having to reoptimize.
ii) Conditional instance normalization
More recently, it has been shown that we can train a network that take both the content image and a discrete style representation as inputs:
f(xc,s)=arg max⁡xL(x∣s,xc)f(\bold{x}_c, s)=\argmax_\bold{x}\mathcal{L}(\bold{x}|s,\bold{x}_c)f(xc​,s)=xargmax​L(x∣s,xc​)
This avoids training a separate network for every style instances.
The key idea is to standardize the features at a given layer using scale and shift parameters that are specific to the style.
In particular, we use conditional instance normalization:
CIN(ϕ(xc),s)=γs(ϕ(xc)−μ(ϕ(xc))σ(ϕ(xc)))+βs\mathrm{CIN}(\phi(\bold{x}_c),s)=\gamma_s\Big(\frac{\phi(\bold{x}_c)-\mu(\phi(\bold{x}_c))}{\sigma(\phi(\bold{x}_c))}\Big)+\beta_sCIN(ϕ(xc​),s)=γs​(σ(ϕ(xc​))ϕ(xc​)−μ(ϕ(xc​))​)+βs​
where μ(ϕ(xs))\mu(\phi(\bold{x}_s))μ(ϕ(xs​))﻿ and σ(ϕ(xs))\sigma(\phi(\bold{x}_s))σ(ϕ(xs​))﻿ are the mean and std of feature maps at a given layer, and γs\gamma_sγs​﻿ and βs\beta_sβs​﻿ are parameters for style sss﻿.
This simple trick is enough to capture many kind of styles.
iii) Adaptative instance normalization
The drawback of the above techniques is that they are limited to a set of pre-defined styles.
Adaptative instance normalization proposes to generalize this by replacing γs\gamma_sγs​﻿ and βs\beta_sβs​﻿ by learned parameters from another network, taking arbitrary style image as input:

The main issue with MLE and ERM is that they pick parameters by minimizing the loss on the training set, which can result in overfitting.
With enough parameters, a model can match any empirical distribution. However, in most cases the empirical distribution is not the same as the true distribution. Regularization add a penalty term to the NLL.
L(θ,λ)=1N∑i=1Nℓ(yn,θ,xn)+λC(θ)=−1N∑i=1Nlog⁡p(yn∣xn,θ)−λlog⁡p(θ)\begin{align} \mathcal{L}(\theta,\lambda)&=\frac{1}{N}\sum^N_{i=1}\ell(y_n,\theta,x_n)+\lambda C(\theta) \\ &=-\frac{1}{N}\sum^N_{i=1} \log p(y_n|x_n,\theta) -\lambda \log p(\theta) \end{align} L(θ,λ)​=N1​i=1∑N​ℓ(yn​,θ,xn​)+λC(θ)=−N1​i=1∑N​logp(yn​∣xn​,θ)−λlogp(θ)​​
by taking the log loss and the log prior for penalty term.
This is equivalent to minimizing the MAP:
θ^map=arg max⁡θlog⁡p(θ∣D)=arg max⁡θ[log⁡p(D∣θ)+log⁡p(θ)−const]\hat{\theta}_{map}=\argmax_{\theta} \log p(\theta|\mathcal{D})=\argmax_{\theta} [\log p(\mathcal{D}|\theta)+\log p(\theta) -\mathrm{const}]θ^map​=θargmax​logp(θ∣D)=θargmax​[logp(D∣θ)+logp(θ)−const]

If we observe just one head, θmle=1\theta_{mle}=1θmle​=1﻿. We add a penalty to θ\thetaθ﻿ to discourage extreme values.

Regularization combines empirical data and prior knowledge.

4.5.2 MAP for MVN

If we set S˘=Ndiag(Σ^mle)\breve{S}=N \mathrm{diag}(\hat{\Sigma}_{mle})S˘=Ndiag(Σ^mle​)﻿:

Off-diagonal elements are shrunk toward 0. λ\lambdaλ﻿ can be chosen via cross-validation of via close form (implemented in scikit-learn LedoitWolf).
The benefits of the MAP is to better behaved than MLE, with eigenvalues of the estimate closer to that of the true covariance matrix (the eigenvectors however are unaffected).

4.5.3 Weight decay

We saw that reducing the degree of the polynomial help the regression task. A more general solution is to add regularization to the regression coefficients.
w^map=arg min⁡wNLL(w)+λ∣∣w∣∣22\hat{w}_{map}=\argmin_w \mathrm{NLL}(w)+\lambda ||w||_2^2w^map​=wargmin​NLL(w)+λ∣∣w∣∣22​
where a higher λ\lambdaλ﻿ means more pressure toward low coefficients and thus less flexibility for the model. This is ridge regression.

4.5.7 Using more data

When a model is too simplistic, it fails to improve as the data grow. A overly complex model will end up converging toward the right solution if enough data is provided.

API Overview
torch.tensor behaves like numpy.array
type generalization
>> torch.tensor([False, 1, 2.0]) tensor([0., 1., 2.])
Python
random
torch.randint(low=0, high=10, size=(3,4)) torch.rand((3,4)) torch.randn((3,4))
Python
adding and multiplying scalar
x = torch.tensor([[1,2,3,4], [5,6,7,8]]) print(x * 10) y = x.add(10) print(y)
Python
reshapping
y = torch.tensor([2, 3, 1, 0]) y = y.view(4,1)
Python
squeeze can also be used for reshapping if the dimension to remove has only one item
x = torch.randn(10,1,10) z1 = torch.squeeze(x, 1) #or z2 = x.squeeze(1)
Python
opposite of squeeze is unsqueeze
x = torch.randn(10,10) z1 = x.unsqueeze(0) print(z1.shape) # torch.size(1,10,10)
Python
the same can be obtained by adding fake dim
z2, z3, z4 = x[None], x[:,None], x[:,:,None] print(z2.shape, z3.shape, z4.shape) # torch.Size([1, 10, 10]) # torch.Size([10, 1, 10]) # torch.Size([10, 10, 1])

Auto gradient of tensor objects
The requires_grad parameter specifies that the gradient is to be calculated for the tensor object
x = torch.tensor([[2., -1.], [1., 1.]], requires_grad=True)
Python
Defining our function. Its gradient is 2 * x
out = x.pow(2).sum()
Python
compute the gradient
out.backward()
Python
We are now in a position to obtain the gradient of out with respect to x
out.grad # tensor([[4., 2.], # [2., 2.]])
Python
Torch vs Numpy
Register tensor objects, aka storing information in a device
x = torch.rand(1, 6400) y = torch.rand(6400, 5000) device = 'cuda' if torch.cuda.is_available() else 'cpu' x, y = x.to(device), y.to(device)
Python
GPU matmul: fast
%timeit z = (x @ y) # It takes 0.515 milli seconds on an average to # perform matrix multiplication
Python
CPU matmul: 20x slower
x, y = x.cpu(), y.cpu() %timeit z = (x @ y) # It takes 9 milli seconds on an average to # perform matrix multiplication
Python
Numpy CPU matmul: 40x slower
x = np.random.random((1, 6400)) y = np.random.random((6400, 5000)) %timeit z = np.matmul(x, y) # It takes 19 milli seconds on an average to # perform matrix multiplication
Python
Building a neural net
Define input and output as
x = [[1, 2], [3, 4], [5, 6], [7, 8]] y = [[3], [7], [11], [15]] device = "cuda" if torch.cuda.is_available() else "cpu" X = torch.tensor(x).float().to(device) Y = torch.tensor(y).float().to(device)
Python
Define module as
must call super in __init__ and overwrite forward
import torch.nn as nn class MyModule(nn.Module): def __init__(self): super().__init__() self.input_to_hidden = nn.Linear(2, 8) self.activation = nn.ReLU() self.hidden_to_class = nn.Linear(8, 1) def forward(self, x): x = self.input_to_hidden(x) x = self.activation(x) x = self.hidden_to_class(x) return x
Python
Instantiate and check weights
my_module = MyModule().to(device) my_module.input_to_hidden.weight #Parameter containing: #tensor([[-0.1474, 0.0559], # [ 0.0454, 0.3088], # [-0.1240, -0.5073], # [-0.3796, 0.6139], # [ 0.0052, -0.6587], # [ 0.0177, 0.5601], # [-0.0376, -0.6641], # [-0.6778, 0.6188]], requires_grad=True) mymodule.parameters() # all parameters
Python
Parameter can replace any tensor creator in __init__
self.input_to_hidden = nn.Parameter(torch.rand(2,8)) # and in forward x = x @ self.input_to_hidden
Python
Define loss function that we optimize for
PyTorch convention: in the loss function, always put the prediction first and then the ground truth
loss_func = nn.MSELoss() # also popular # nn.CrossEntropyLoss, for categorical # nn.BCELoss, for binary Y_hat = mymodule(X) print(loss_func(Y_hat, Y))
Python
Custom loss can also be defined
def my_MAELoss(y_hat, y): loss = (y_hat - y).abs() loss = loss.mean() return loss
Python
Define optimizer
from torch.optim import SGD opt = SGD(mymodule.parameters(), lr=.001)
Python
Wrapup
reset gradient at each epoch with zero_grad
mymodule = MyModule() opt = nn.SGD(mymodule.parameters(), lr=.001) loss_history = [] for idx in range(n_epoch): opt.zero_grad() Y_hat = mymodule(X) loss_value = loss_func(Y_hat, Y) loss_value.backward() opt.step() loss_history.append(loss_value)
Python
Dataset, DataLoader, and batch size
Batch size is the number of point we consider to compute the loss function
representative enough of the dataset, and memory efficient
Implement dataset class
import torch import torch.nn as nn from torch.utils.data import DataSet, DataLoader class MyDataSet(DataSet): def __init__(self, x, y): self.X = torch.tensor(x).float() self.Y = torch.tensor(y).float() def __getitem__(self, idx): return self.X[idx], self.Y[idx] def __len__(self): return len(self.X)
Python
Use it with dataloader
ds = MyDataSet(x, y) dl = DataLoader(ds, batch_size=2, shuffle=True) for _ in range(n_epoch): for x, y in dl: ...

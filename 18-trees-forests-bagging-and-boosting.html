18.1 Classification and regression trees

Trees consist of a set of nested rules: at each node j, the feature djd_jdj​﻿ of x\bold{x}x﻿ is compared to a threshold tjt_jtj​﻿: if it’s lower, the input flows to the left branch, otherwise to the right.
At the leaves, the model predicts the output
A region can be defined as: R1={x:x1≤t1,x2≤t2}R_1=\{\bold{x}:x_1\leq t_1,x_2 \leq t_2\}R1​={x:x1​≤t1​,x2​≤t2​}﻿​
We can associate a mean response to each region, resulting in a piecewise constant surface
Outputs can be estimated using:
wj=∑n=1NynI(xn∈Rj)∑n=1NI(xn∈Rj)w_j=\frac{\sum_{n=1}^Ny_n \mathbb{I}(\bold{x_n}\in R_j)}{\sum_{n=1}^N\mathbb{I}(\bold{x_n}\in R_j)}wj​=∑n=1N​I(xn​∈Rj​)∑n=1N​yn​I(xn​∈Rj​)​﻿​
A regression tree can be defined as: f(x,θ)=∑j=1JwjI(xj∈Rj)f(\bold{x},\bold{\theta})=\sum_{j=1}^Jw_j \mathbb{I}(\bold{x}_j\in R_j)f(x,θ)=∑j=1J​wj​I(xj​∈Rj​)﻿,
with θ={(Rj,wj):j=1:J}\bold{\theta}=\{(R_j,w_j): j=1:J\}θ={(Rj​,wj​):j=1:J}﻿​
For classification problems, a leaf contains a distribution of the label rather than a mean.
To fit the model, we need to minimize the loss:
L(θ)=∑nl(yn,f(xn,θ))=∑j=1J∑xj∈Rjl(yn,wj)\mathcal{L}(\bold{\theta})=\sum_nl(y_n,f(\bold{x_n},\bold{\theta}))= \sum_{j=1}^J\sum_{x_j \in R_j}l(y_n, w_j)L(θ)=∑n​l(yn​,f(xn​,θ))=∑j=1J​∑xj​∈Rj​​l(yn​,wj​)﻿​
Finding a discrete tree structure is non-differentiable and finding the optimal partition is NP-complete. We use a greedy procedure to grow one node at a time (CART, C4.5, ID3 methods)

2 regularization approaches to limit too small partitions and overfitting:
Limit the tree depth by some heuristic: max depth or minimum number of values in leaves
Grow the tree to its maximum depth, and then prune it back, by merging subtrees back to their parents (more computationally expensive)
Pros of trees:
Easy to interpret
Handle a mix of numerical and categorical variables
Relatively robust to outliers
Fast to build and scale well to large datasets
Handle missing values
Cons:
Relatively unstable: small change of one data point can change the surface decision a lot (high variance)

18.2 Ensemble learning

Ensemble is a way to reduce variance (but similar bias): f(y∣x)=1M∑mfm(y∣x)f(y|\bold{x})=\frac{1}{\mathcal{M}}\sum_mf_m(y|\bold{x})f(y∣x)=M1​∑m​fm​(y∣x)﻿​
Averaging for regression
Classification can also use a majority vote (committee method), by assuming each predictor to make independent errors
Stacking add weights to the average: f(y∣x)=1M∑mwmfm(y∣x)f(y|\bold{x})=\frac{1}{\mathcal{M}}\sum_mw_mf_m(y|\bold{x})f(y∣x)=M1​∑m​wm​fm​(y∣x)﻿, needs to be trained on an external dataset, otherwise all the weight will be put on the most performing model

18.3 Bagging

Bagging (bootstrap aggregating) create M learners by sampling with replacement N data points from the original dataset (of size N) to generalize and stabilize the decision process.
The disadvantage is that on average, only 63% of the original data is sampled. The 37% out-of-bag dataset can be used as a validation set: the probability of a point not to be selected is limN→∞(1−1/N)N=e−1=0.37lim_{N\rightarrow \infin}(1-1/N)^N=e^{-1}=0.37limN→∞​(1−1/N)N=e−1=0.37﻿​
Bagging doesn't always improve performance: it relies on the instability of the sub-models when supplied with fewer data. It works with decision trees but less with DNN that will underperform

18.4 Random forests

In addition to bagging, random forests also generate a random feature subset SiS_iSi​﻿ to decorrelate the base learners even more.
This fig shows that RF works best than bagging alone with the HP dataset containing many irrelevant features for spam detection.
Boosting works even better, but relies on a sequential fitting of trees, whereas RFs can be fit in parallel.

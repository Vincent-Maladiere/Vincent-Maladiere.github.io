17.2 Gaussian Processes

17.2 Gaussian processes

Gaussian processes is a way to define distributions over functions of the form f:X→Rf:\mathcal{X}\rightarrow \Rf:X→R, where X\mathcal{X}X is any domain.

The key assumption is that the function values at a set of MMM inputs, [f(x1),…,f(xM)][f(\bold{x}_1),\dots,f(\bold{x}_M)][f(x1​),…,f(xM​)], is jointly Gaussian, with mean μ=[m(x1),…,m(xM)]\mu=[m(\bold{x}_1),\dots,m(\bold{x}_M)]μ=[m(x1​),…,m(xM​)] and covariance Σij=K(xi,xj)\Sigma_{ij}=\mathcal{K}(\bold{x}_i,\bold{x}_j)Σij​=K(xi​,xj​), where mmm is a mean function and K\mathcal{K}K is a Mercer kernel.

This assumption holds when M=N+1M=N+1M=N+1, containing NNN training points xn\bold{x}_nxn​ and 111 test point x∗\bold{x}_*x∗​.

Thus, we can infer f(x∗)f(\bold{x}_*)f(x∗​) from knowledge of f(x1),…,f(xn)f(\bold{x}_1),\dots,f(\bold{x}_n)f(x1​),…,f(xn​) by manipulating the joint Gaussian distribution p(f(x1),…,f(xn),f(x∗))p(f(\bold{x}_1),\dots,f(\bold{x}_n),f(\bold{x}_*))p(f(x1​),…,f(xn​),f(x∗​)).

We can also extend this to work with noisy observations, such as in regression or classification problems.

17.2.1 Noise-free observations

Suppose we observe a training set D={(xn,yn):n=1:N}\mathcal{D}=\{(\bold{x}_n,y_n):n=1:N\}D={(xn​,yn​):n=1:N}, where yn=f(xn)y_n=f(\bold{x}_n)yn​=f(xn​) is the noise-free function at xn\bold{x}_nxn​.

We want the GP to act as an interpolator of the training data, i.e. returning the answer f(x)f(\bold{x})f(x) with no uncertainty when it has already seen x\bold{x}x.

Let’s now consider a test set X∗X_*X∗​ of size N∗×DN_*\times DN∗​×D (not in D\mathcal{D}D). We want to predict the function outputs f∗=[f(x1),…,f(xN∗)]f_*=[f(\bold{x}_1),\dots,f(\bold{x}_{N_*})]f∗​=[f(x1​),…,f(xN∗​​)].

By definition of the GP, the joint distribution p(fX,f∗∣X,X∗)p(f_X,f_*|X,X_*)p(fX​,f∗​∣X,X∗​) has the form:

[fXf∗]=N([μXμ∗],[KX,XKX,∗KX,∗⊤K∗,∗])\begin{bmatrix} f_X \\ f_* \end{bmatrix}=\mathcal{N} \Bigg( \begin{bmatrix} \mu_X \\ \mu_* \end{bmatrix}, \begin{bmatrix} K_{X,X} & K_{X,*} \\ K_{X,*}^\top & K_{*,*} \end{bmatrix} \Bigg)[fX​f∗​​]=N([μX​μ∗​​],[KX,X​KX,∗⊤​​KX,∗​K∗,∗​​])

where μX=[m(x1),…,m(xN)]\mu_X=[m(\bold{x}_1),\dots,m(\bold{x}_N)]μX​=[m(x1​),…,m(xN​)], μ∗=[m(x1∗),…,m(xN∗)]\mu_*=[m(\bold{x}_1^*),\dots,m(\bold{x}_N^*)]μ∗​=[m(x1∗​),…,m(xN∗​)], and KX,X=K(X,X)K_{X,X}=\mathcal{K}(X,X)KX,X​=K(X,X) is N×NN\times NN×N.

By the standard rule for conditioning Gaussians, the posterior distribution is:

p(f∗∣X∗,D)=N(f∗∣μ∗,Σ∗)μ∗=m(X∗)+KX,∗⊤KX,X−1(fX−m(X))Σ∗=K∗,∗−KX,∗⊤KX,X−1KX,∗\begin{align} p(f_*|X_*,\mathcal{D}) &=\mathcal{N}(f_*|\mu_*,\Sigma_*) \\ \mu_* &= m(X_*)+K_{X,*}^\top K_{X,X}^{-1}(f_X-m(\bold{X})) \\ \Sigma_* &= K_{*,*}-K_{X,*}^\top K_{X,X}^{-1}K_{X,*} \end{align}p(f∗​∣X∗​,D)μ∗​Σ∗​​=N(f∗​∣μ∗​,Σ∗​)=m(X∗​)+KX,∗⊤​KX,X−1​(fX​−m(X))=K∗,∗​−KX,∗⊤​KX,X−1​KX,∗​​​

We see the uncertainty increase as we move further away from the training points.

17.2.2 Noisy observations

Let’s now consider the case where we observe a noisy version of the underlying function:

yn=f(xn)+ϵny_n=f(\bold{x}_n)+\epsilon_nyn​=f(xn​)+ϵn​

where ϵ∼N(0,σy2)\epsilon \sim \mathcal{N}(0,\sigma_y^2)ϵ∼N(0,σy2​).

In this case, the model is not required to interpolate the training data, but it must come “close”.

The covariance of the noisy response is:

Cov[yi,yj]=Cov[fi,fj]+Cov[ϵi,ϵj]=K(xi,xj)+σy2δi,j\mathrm{Cov}[y_i,y_j]=\mathrm{Cov}[f_i,f_j]+\mathrm{Cov}[\epsilon_i,\epsilon_j]=\mathcal{K}(\bold{x}_i,\bold{x}_j)+\sigma^2_y \delta_{i,j}Cov[yi​,yj​]=Cov[fi​,fj​]+Cov[ϵi​,ϵj​]=K(xi​,xj​)+σy2​δi,j​

In other words:

Cov[y∣X]=KX,X+σy2IN≜Kσ\mathrm{Cov}[y|X]=K_{X,X}+\sigma^2_yI_N\triangleq K_{\sigma}Cov[y∣X]=KX,X​+σy2​IN​≜Kσ​

The density of the observed data and the latent, noise-free function on the test points is:

[yf∗]=N([μXμ∗],[KσKX,∗KX,∗⊤K∗,∗])\begin{bmatrix} y\\ f_* \end{bmatrix}=\mathcal{N} \Bigg( \begin{bmatrix} \mu_X \\ \mu_* \end{bmatrix}, \begin{bmatrix} K_{\sigma} & K_{X,*} \\ K_{X,*}^\top & K_{*,*} \end{bmatrix} \Bigg)[yf∗​​]=N([μX​μ∗​​],[Kσ​KX,∗⊤​​KX,∗​K∗,∗​​])

Hence the posterior predictive density at a set of points X∗X_*X∗​ is:

p(f∗∣X∗,D)=N(f∗∣μ∗,Σ∗)μ∗∣X=μ∗+KX,∗⊤Kσ−1(y−μX))Σ∗∣X=K∗,∗−KX,∗⊤Kσ−1KX,∗\begin{align} p(f_*|X_*,\mathcal{D}) &=\mathcal{N}(f_*|\mu_*,\Sigma_*) \\ \mu_{*|X} &= \mu_*+K_{X,*}^\top K_{\sigma}^{-1}(y-\mu_X)) \\ \Sigma_{*|X} &= K_{*,*}-K_{X,*}^\top K_{\sigma}^{-1}K_{X,*} \end{align}p(f∗​∣X∗​,D)μ∗∣X​Σ∗∣X​​=N(f∗​∣μ∗​,Σ∗​)=μ∗​+KX,∗⊤​Kσ−1​(y−μX​))=K∗,∗​−KX,∗⊤​Kσ−1​KX,∗​​​

In the case of a single test point x∗\bold{x}_*x∗​, this simplifies as follow:

μ∗∣X=m(x∗)+kX,∗⊤Kσ−1(y−μX)Σ∗∣X=k∗,∗−kX,∗⊤Kσ−1kX,∗\begin{align} \mu_{*|X}&=m(\bold{x}_*)+k_{X,*}^\top K_\sigma^{-1}(y-\mu_X) \\ \Sigma_{*|X}&=k_{*,*}-k^\top_{X,*}K_\sigma^{-1}k_{X,*} \end{align}μ∗∣X​Σ∗∣X​​=m(x∗​)+kX,∗⊤​Kσ−1​(y−μX​)=k∗,∗​−kX,∗⊤​Kσ−1​kX,∗​​​

where k∗,∗=K(x∗,x∗)k_{*,*}=\mathcal{K}(\bold{x}_*,\bold{x}_*)k∗,∗​=K(x∗​,x∗​) and kX,∗=[K(x1,x∗),…,K(xN,x∗)]k_{X,*}=[\mathcal{K}(\bold{x}_1,\bold{x}_*),\dots,\mathcal{K}(\bold{x}_N,\bold{x}_*)]kX,∗​=[K(x1​,x∗​),…,K(xN​,x∗​)].

If the mean function is zero, we can write the posterior mean as:

μ∗∣X=kX,∗⊤(Kσ−1y)≜kX,∗⊤α=∑n=1NK(xn,x∗)αn\mu_{*|X}=k_{X,*}^\top (K_{\sigma}^{-1}y)\triangleq k_{X,*}^\top \alpha=\sum_{n=1}^N \mathcal{K}(\bold{x}_n,\bold{x_*})\alpha_nμ∗∣X​=kX,∗⊤​(Kσ−1​y)≜kX,∗⊤​α=n=1∑N​K(xn​,x∗​)αn​

This is identical to the predictions from kernel ridge regression.

17.2.3 Comparison to kernel regression

In section 16.3.5 we discussed kernel regression, which is a generative approach to regression in which we approximate p(y,x)p(y,\bold{x})p(y,x) using a density kernel estimation.

E[y∣x,D]=∑n=1Nynwn(x)wn(x)=Kh(x−xn)∑n′=1NKh(x−xn′)\begin{align} \mathbb{E}[y|\bold{x},\mathcal{D}]&=\sum_{n=1}^N y_n w_n(\bold{x}) \\ w_n(\bold{x})&=\frac{\mathcal{K}_h(\bold{x}-\bold{x}_n)}{\sum_{{n'}=1}^N \mathcal{K}_h(\bold{x}-\bold{x}_{n'})} \end{align}E[y∣x,D]wn​(x)​=n=1∑N​yn​wn​(x)=∑n′=1N​Kh​(x−xn′​)Kh​(x−xn​)​​​

This is very similar to our previous posterior mean μ∗∣X\mu_{*|X}μ∗∣X​, we few important differences:

  1. In a GP, we use Mercer kernel instead of density kernel. Mercer kernels can be defined on objects such as string or graphs, which is harder to do for density kernels.
  2. GP is an interpolator (when σy2=0\sigma^2_y =0σy2​=0) , so E[y∣xn,D]=yn\mathbb{E}[y|\bold{x}_n,\mathcal{D}]=y_nE[y∣xn​,D]=yn​. By contrast, kernel regression is not an interpolator..
  3. GP is a Bayesian model, meaning we can estimate hyperparameters of the kernel by maximizing the marginal likelihood. By contrast, in kernel regression we must use cross-validation to estimate the kernel parameters, such as bandwidth.
  4. Computing the weights in kernel regression takes O(N)O(N)O(N) times, whereas computing the weigths αn\alpha_nαn​ for GP regression takes O(N3)O(N^3)O(N3) time (although there are approximations that can reduce it to MN2)MN^2)MN2)

Also note that Bayesian linear regression is a special case of a GP. The former works in weight space whereas the latter works in function space.

17.2.6 Estimating the kernel

Most models have some free parameters that can play a great role on the predictions.

Let assume we perform a 1d regression using GP with a RBF kernel:

K(xi,xj)=σf2exp⁡(−(xi−xj)22ℓ2)\mathcal{K}(x_i,x_j)=\sigma_f^2\exp\Big(-\frac{(x_i-x_j)^2}{2\ell^2}\Big)K(xi​,xj​)=σf2​exp(−2ℓ2(xi​−xj​)2​)

ℓ\ellℓ is the horizontal scale over which the function changes, σf2\sigma^2_fσf2​ control the vertical scale. We assume observation noise with variance σy2\sigma_y^2σy2​.

We sampled 20 points from an MVN with covariance given by Σ=K(xi,xj)\Sigma=\mathcal{K}(x_i,x_j)Σ=K(xi​,xj​) for a grid of points {xi}\{x_i\}{xi​}, and added observation noise of value σy.\sigma_y.σy​.

We then fit these points using a GP with the same kernel, but with a range of hyperparameters.

When changing ℓ\ellℓ from 1 to 3, the function goes from a good fit to very smooth.

17.2.6.1 Empirical Bayes

To estimate the kernel parameters θ\thetaθ (aka hyperparameters), we could use exhaustive grid search with the validation loss as an objective, but this can be slow (this is the approach used by nonprobabilistic methods such as SVMs).

We consider here the empirical Bayes approach, which use gradient-based approach, which are faster. We maximize the marginal likelihood (and not likelihood, because we marginalized out the latent Gaussian vector fff):

p(y∣X,θ)=∫p(y∣f,X)p(f∣X,θ)dfp(\bold{y}|X,\theta)=\int p(\bold{y}|f,X)p(f|X,\theta)dfp(y∣X,θ)=∫p(y∣f,X)p(f∣X,θ)df

Assuming the mean is 0, we have:

p(f∣X,θ)=N(f∣0,K)p(y∣f,X)=∏n=1NN(yn∣fn,σy2)\begin{align} p(f|X,\theta) &=\mathcal{N}(f|0,K) \\ p(\bold{y}|f,X) &=\prod_{n=1}^N\mathcal{N}(y_n|f_n, \sigma^2_y) \end{align}p(f∣X,θ)p(y∣f,X)​=N(f∣0,K)=n=1∏N​N(yn​∣fn​,σy2​)​​

So the marginal likelihood is:

log⁡p(y∣X,θ)=log⁡N(y∣0,Kσ)=−12y⊤Kσ−1y−12log⁡∣Kσ∣−N2log⁡(2π)\log p(\bold{y}|X,\theta)= \log \mathcal{N}(\bold{y}|0,K_\sigma)= -\frac{1}{2}\bold{y}^\top K_\sigma^{-1} \bold{y}-\frac{1}{2}\log |K_\sigma|-\frac{N}{2}\log(2\pi )logp(y∣X,θ)=logN(y∣0,Kσ​)=−21​y⊤Kσ−1​y−21​log∣Kσ​∣−2N​log(2π)

with the dependence of Kσ=KX,X+σy2INK_\sigma=K_{X,X}+\sigma^2_yI_NKσ​=KX,X​+σy2​IN​ on θ\thetaθ implicit.

The first term is a data fit term, the second is a model complexity term (the third is just a constant).

If the length scale ℓ\ellℓ is small, the fit will be rather good so y⊤Kσ−1y\bold{y}^\top K_\sigma^{-1}\bold{y}y⊤Kσ−1​y will be small. However, KKK will be almost diagonal since most points will not be considered “near” any others, so log⁡∣Kσ∣\log |K_\sigma|log∣Kσ​∣ will be large.

We now maximize the marginal likelihood:

∂∂θjlog⁡(y∣X,θ)=−12y⊤Kσ−1∂Kσ∂θjKσ−1y−12tr(Kσ−1∂Kσ∂θj)=12tr((αα⊤−Kσ−1)∂Kσ∂θj)\begin{align} \frac{\partial}{\partial \theta_j}\log (\bold{y}|X,\theta)&=-\frac{1}{2}\bold{y}^\top K_\sigma^{-1}\frac{\partial K_\sigma}{\partial \theta_j} K_\sigma^{-1} \bold{y}-\frac{1}{2}\mathrm{tr}(K_\sigma^{-1}\frac{\partial K_\sigma}{\partial \theta_j}) \\ &= \frac{1}{2}\mathrm{tr}\Bigg((\alpha \alpha^\top -K_\sigma^{-1})\frac{\partial K_\sigma}{\partial \theta_j}\Bigg) \end{align}∂θj​∂​log(y∣X,θ)​=−21​y⊤Kσ−1​∂θj​∂Kσ​​Kσ−1​y−21​tr(Kσ−1​∂θj​∂Kσ​​)=21​tr((αα⊤−Kσ−1​)∂θj​∂Kσ​​)​​

where α=y⊤Kσ−1\alpha=\bold{y}^\top K_\sigma^{-1}α=y⊤Kσ−1​

It takes O(N3)O(N^3)O(N3) time to compute Kσ−1K_\sigma^{-1}Kσ−1​, and then O(N2)O(N^2)O(N2) time per hyperparameter to compute the gradient.

The form of ∂Kσ∂θj\frac{\partial K_\sigma}{\partial \theta_j}∂θj​∂Kσ​​ depends on the form of the kernel, and which parameters we are taking derivatives with respect to. We often have constraints on the HP such as σy2≥0\sigma^2_y\geq0σy2​≥0. In this case, we can define θ=log⁡(σy2)\theta=\log(\sigma^2_y)θ=log(σy2​) and then use the chain rule.

Given an expression for the log marginal likelihood and its derivative, we can estimate the kernel parameters with any gradient-based optimizer. However, since the objective is not convex, local minima can be a problem so we need multiple restart.

Notice that when the noise is high (around σy2=1\sigma_y^2=1σy2​=1), the marginal likelihood is insensitive to the length scale. Similarly, when the length scale is very short ℓ=0.5\ell=0.5ℓ=0.5, the kernel interpolates exactly the data.

17.2.6.2 Bayesian inference

When we have a small number of data points (e.g. using GP for Bayesian optimization) using a point estimate of the kernel parameters can give poor results.

Instead, we can approximate the posterior distribution over the parameters, using slice sampling, Hamiltonian Monte Carlo or sequential Monte Carlo.

17.2.7 GPs for classification

So far, we have used GPs for regression, which uses Gaussian likelihood. In this case, the posterior is also a GP and all the computation can be performed analytically.

However, if the likelihood is non Gaussian, such as the Bernoulli likelihood for binary classification, we can no longer compute the posterior exactly.

There are various approximation we can make, and we use there the Hamiltonian Monte Carlo method, both for the latent Gaussian fff and the kernel parameters θ\thetaθ.

We specify the negative log joint:

−E(f,θ)=log⁡p(f,θ∣X,y)=log⁡N(f∣0,KX,X)+∑n=1Nlog⁡Ber(yn∣fn(xn))+log⁡(θ)\begin{align} -\mathcal{E}(f,\theta)&=\log p(f,\theta|X,\bold{y})\\ &=\log \mathcal{N}(f|0,K_{X,X})+\sum_{n=1}^N \log \mathrm{Ber}(y_n|f_n(\bold{x}_n))+\log(\theta) \end{align}−E(f,θ)​=logp(f,θ∣X,y)=logN(f∣0,KX,X​)+n=1∑N​logBer(yn​∣fn​(xn​))+log(θ)​​

We then use autograd to compute ∇fE(f,θ)\nabla_f\mathcal{E}(f,\theta)∇f​E(f,θ) and ∇θE(f,θ)\nabla_\theta\mathcal{E}(f,\theta)∇θ​E(f,θ), and use these gradients as input to a Gaussian proposal distribution.

We consider a 1d example of binary classification:

The SE kernel curves probability toward 0.5 on the edges, because the prior mean function has m(x)=0m(x)=0m(x)=0 and σ(0)=0.5\sigma(0)=0.5σ(0)=0.5.

We can eliminate this artifact by using a more flexible kernel that encodes the knowledge that the probability is monotonically increasing or decreasing. We can use a linear kernel, scale it and add it to the SE kernel:

K(x,x′)=τ(x−c)(x′−c′)+exp⁡[−(x−x′)22ℓ2]\mathcal{K}(x,x')=\tau(x-c)(x'-c')+\exp\Big[-\frac{(x-x')^2}{2\ell^2}\Big]K(x,x′)=τ(x−c)(x′−c′)+exp[−2ℓ2(x−x′)2​]

17.2.9 Scaling DPs to large datasets

The main issue with GPs (and kernel methods such as SVM) is that it requires inverting the N×NN\times NN×N kernel matrix, which takes O(N3)O(N^3)O(N3) times, making the method too slow for big datasets.

We review some speed-up techniques.

17.2.9.1 Sparse (inducing-point) approximations

A simple approach to speeding up GPs is to use less data. A better approach is to summarize the NNN training points into M≪NM\ll NM≪N inducing points or pseudo inputs ZZZ.

This lets us replace p(f∣fX)p(f|f_X)p(f∣fX​) with p(f∣fZ)p(f|f_Z)p(f∣fZ​) where fX={f(x):x∈X}f_X=\{f(\bold{x}):\bold{x}\in X\}fX​={f(x):x∈X} is the vector of observed function values at training points.

By estimating (Z,fz)(Z,f_z)(Z,fz​) we can learn to compress the training data and speed up the computation to O(M3)O(M^3)O(M3). This is called a sparse GP. This whole process can be performed using the framework of variational inference.

17.2.9.2 Exploiting parallelization and kernel matrix structure

It takes O(N3)O(N^3)O(N3) to compute the Cholesky decomposition of KX,XK_{X,X}KX,X​, which is needed to solve the linear system Kσα=yK_\sigma \alpha=\bold{y}Kσ​α=y and compute ∣KX,X∣|K_{X,X}|∣KX,X​∣.

An alternative is to use linear algebra methods, often called Krylov subspace methods, which are based on matrix vector multiplication (MVM). These approaches are usually faster because they exploit structure in KKK.

Moreover, even if the Kernel matrix doesn’t have a special structure, matrix multiplication can easily be parallelized over GPUs, unlike Cholesky based methods which are sequential.

This is the basis of the GPyTorch package.

17.2.9.3 Random feature approximation

We can approximate the feature map for many shift invariant kernels using a randomly chosen finite set of MMM basis functions, thus reducing the cost to O(NM+M3)O(NM+M^3)O(NM+M3).

i) Random features for RBF kernel

For Gaussian RBF kernel, one can show that:

K(x,x′)≈ϕ(x′)⊤ϕ(x′)\mathcal{K}(\bold{x,x'})\approx\phi(\bold{x'})^\top\phi(\bold{x'})K(x,x′)≈ϕ(x′)⊤ϕ(x′)

where the features vector are:

ϕ(x)=1T[sin⁡(ω1⊤x),…,sin⁡(ωT⊤x),cos⁡(ω1⊤x),…,cos⁡(ωT⊤x)]=1T[sin⁡(Ωx),cos⁡(Ωx)]\begin{align} \phi(\bold{x})&=\frac{1}{\sqrt{T}}[\sin(\bold{\omega}_1^\top \bold{x}),\dots,\sin(\bold{\omega}_T^\top \bold{x}),\cos(\bold{\omega}_1^\top \bold{x}),\dots,\cos(\bold{\omega}_T^\top \bold{x})] \\ &= \frac{1}{\sqrt{T}}[\sin(\Omega \bold{x}),\cos(\Omega \bold{x})] \end{align}ϕ(x)​=T​1​[sin(ω1⊤​x),…,sin(ωT⊤​x),cos(ω1⊤​x),…,cos(ωT⊤​x)]=T​1​[sin(Ωx),cos(Ωx)]​​

where T=M/2T=M/2T=M/2 and Ω∈RT×D\Omega\in \R^{T\times D}Ω∈RT×D is a random Gaussian matrix, where the entries are sampled iid from N(0,1/σ2)\mathcal{N}(0,1/\sigma^2)N(0,1/σ2) where σ\sigmaσ is the kernel bandwidth.

The bias of the approximation decreases as MMM increases. In practice, we compute a single sample Monte Carlo approximation to the expectation by drawing a single random matrix.

The features in the equation above are called random Fourier features (RFF).

We can also use positive random features instead of trigonometric random features, which can be preferable for some applications, like models using attention. We can use:

ϕ(x)≜1Me−∣∣x∣∣2/2[exp⁡(ω1⊤x),…,exp⁡(ωM⊤x)]\phi(\bold{x})\triangleq \frac{1}{\sqrt{M}}e^{-||\bold{x}||^2/2}[\exp(\omega_1^\top\bold{x}),\dots,\exp(\omega_M^\top\bold{x})]ϕ(x)≜M​1​e−∣∣x∣∣2/2[exp(ω1⊤​x),…,exp(ωM⊤​x)]

Regardless of the choice of the features, we can obtain a lower variance estimate by ensuring the rows of ZZZ are random but orthogonal. These are called orthogonal random features.

Such sampling can be conducted efficiently via Gram-Schmidt orthogonalization of the unstructured Gaussian matrices.

ii) Fastfood approximation

Storing the random matrix Ω\OmegaΩ takes O(MD)O(MD)O(MD) space and computing Ωx\Omega \bold{x}Ωx takes O(MD)O(MD)O(MD) time, where DDD is the input dimensionality and MMM is the number of random features.

This can become prohibitive when M≫DM\gg DM≫D, which it may needs to get any benefits over using the original set of features.

Fortunately, we can use the fast Hadamard transform to reduce the memory to O(M)O(M)O(M) and reduce the time to O(Mlog⁡D)O(M\log D)O(MlogD). This approach has been called fastfood as reference to the original term “kitchen sinks”.

iii) Extreme learning machines

We can use the random features approximation to the kernel to convert a GP into a linear model of the form:

f(x,θ)=Wϕ(x)=Wh(Zx)f(\bold{x},\theta)=W\phi(\bold{x})=Wh(Z\bold{x})f(x,θ)=Wϕ(x)=Wh(Zx)

where h(a)=1/M[sin⁡(a),cos⁡(a)]h(a)=\sqrt{1/M}[\sin(a),\cos(a)]h(a)=1/M​[sin(a),cos(a)] for RBF kernels. This is equivalent to a MLP with fixed and random input-to-hidden weights.

When M>NM>NM>N, this corresponds to an over-parametrized model which can perfectly interpolate the training data.

Alternatively, we can use M<NM<NM<N, but stack many nonlinear random layers together and just optimize the output weights. This has been called an extreme learning machine (ELM).

13.4 Training Neural Networks

13.4 Training neural networks

We now discuss how to fit DNNs to data. The standard approach is to use MLE, by minimizing the NLL:

NLL(θ)=−log⁡p(D∣θ)=−∑n=1Nlog⁡p(yn∣xn;θ)\mathrm{NLL}(\theta)=-\log p(\mathcal{D}|\theta)=-\sum_{n=1}^N \log p(y_n|x_n;\theta)NLL(θ)=−logp(D∣θ)=−n=1∑N​logp(yn​∣xn​;θ)

In principle, we can use the backprop algorithm and compute the gradient of this loss and pass it to an optimizer. Adam (section 8.4.6.3) is a popular choice, due to its ability to scale to large datasets (by virtue of being of SGD-type) and converges fairly quickly (by virtue of using diagonal preconditioning and momentum).

However, in practice, this may not work well. In addition to practical issues, there are theoretical limitations. In particular, the loss of DNNs is not convex, so we will generally not find the global optimum.

Nevertheless, SGD can often find surprisingly good solutions.

13.4.1 Tuning the learning rate

It is essential to tune the learning rate, to ensure convergence to a good solution (see section 8.4.3).

13.4.2 Vanishing and exploding gradient

When training very deep networks, the gradient tends to become very small (vanishing) or very large (exploding), because the signal is passed through a series of layers that either amplify it or diminish it.

Consider the gradient of layer lll:

∂L∂zl=∂L∂zl+1∂zl+1∂zl=Jlgl+1\frac{\partial \mathcal{L}}{\partial z_l}= \frac{\partial \mathcal{L}}{\partial z_{l+1}}\frac{\partial z_{l+1}}{\partial z_l}=J_l \bold{g}_{l+1}∂zl​∂L​=∂zl+1​∂L​∂zl​∂zl+1​​=Jl​gl+1​

If JlJ_lJl​ is constant across layers, the contribution of the gradient from the final layer, gL\bold{g}_LgL​, to layer lll will be JL−lgLJ^{L-l}\bold{g}_LJL−lgL​. Thus the behavior of the system depends on the eigenvectors of JJJ.

Although JJJ is a real-valued matrix, it is not symmetric, so its eigenvalues and eigenvectors can be complex-valued, with the imaginary components corresponding to oscillatory behavior.

Let λ\lambdaλ be the spectral radius of JJJ (the largest absolute eigenvalues). If λ>1\lambda> 1λ>1, the gradient can explode, if λ<1\lambda <1λ<1 the gradient can vanish.

We can counter the exploding problem with gradient clipping, in which we cap the value of the magnitude of the gradient to ccc if it becomes too large:

g=min⁡(1,c∣∣g∣∣)g\bold{g}=\min(1,\frac{c}{||\bold{g}||})\bold{g}g=min(1,∣∣g∣∣c​)g

However, the vanishing problem is more difficult to solve. There are various solutions:

  • Modify the activation functions (see next section)
  • Modify the architecture so that the updates are additive rather than multiplicative (residual networks)
  • Modify the architecture to standardize the activation at each layer, so that the distribution of activations over the dataset remains constant during training (batch norm).
  • Carefully choose the initial values of the parameters

13.4.3 Non-saturating activation functions

As already mentioned in section 13.2.3, the sigmoid activation function and tanh⁡\tanhtanh function saturates for small and large values of activations, vanishing gradient.

For the sigmoid of a linear layer, we have:

∂L∂x=W⊤z(1−z)∂L∂W=z(1−z)x⊤\begin{align} \frac{\partial \mathcal{L}}{\partial \bold{x}} &= W^\top z(1-z)\\ \frac{\partial \mathcal{L}}{\partial W} &= z(1-z)\bold{x}^\top \end{align}∂x∂L​∂W∂L​​=W⊤z(1−z)=z(1−z)x⊤​​

Hence, if the activation zzz is close to 0 or 1, the gradient is close to 0.

13.4.3.1 ReLU

The rectified linear unit is defined as:

ReLU(x)=max⁡(0,x)=xI(x>0)\mathrm{ReLU}(x)=\max(0, x)=x\mathbb{I}(x>0)ReLU(x)=max(0,x)=xI(x>0)

Its gradient is:

ReLU′(x)=I(x>0)\mathrm{ReLU}'(x)=\mathbb{I}(x>0)ReLU′(x)=I(x>0)

Therefore, with a linear layer, we have:

∂L∂x=W⊤I(z>0)∂L∂W=I(z>0)x⊤\begin{align} \frac{\partial \mathcal{L}}{\partial \bold{x}}&=W^\top \mathbb{I}(\bold{z}>0) \\ \frac{\partial \mathcal{L}}{\partial W} &= \mathbb{I}(\bold{z}>0) \bold{x}^\top \end{align}∂x∂L​∂W∂L​​=W⊤I(z>0)=I(z>0)x⊤​​

Hence the gradient will not vanish, as long as z\bold{z}z is positive.

However, if some of the weights are initialized with large negative values, then some activations will go to zero and their gradient too. The algorithm will never be able to escape this situation, so some hidden units will stay permanently off. This is called the “dead-ReLU” problem.

13.4.3.2 Non-saturating ReLU

The leaky ReLU has been proposed to solve the dead-ReLU issue:

LRELU(x;α)=max⁡(αx,x)\mathrm{LRELU}({x};\alpha)=\max(\alpha {x},{x})LRELU(x;α)=max(αx,x)

with 0<α<10<\alpha<10<α<1, which allows some signal to be passed back to earlier layers, even when the input is negative.

If we allow the parameter α\alphaα to be learned, this is called parametric ReLU.

Another popular choice is ELU (exponential linear unit):

ELU(x;α)={α(ex−1)if  x≤0xotherwise\mathrm{ELU}(x;\alpha)=\begin{cases}\alpha (e^x-1 )&\mathrm{if}\;x\leq0\\x&\mathrm{otherwise}\end{cases}ELU(x;α)={α(ex−1)x​ifx≤0otherwise​

This has the advantage of being a smooth function.

A slight variant of ELU is known as SELU (self-normalizing ELU):

SELU(x;α,λ)=λELU(x;α)\mathrm{SELU}(x;\alpha,\lambda)=\lambda \mathrm{ELU}(x;\alpha)SELU(x;α,λ)=λELU(x;α)

Surprisingly, it has been proven that by setting α\alphaα and λ\lambdaλ carefully, the activation function ensures that the output of each layer is standardized (providing the input is also standardized), even without the use of techniques such as batch norm.

13.4.3.3 Other choices

Switch or SiLU (sigmoid rectified unit) appears to work well for image classification benchmarks and is defined as:

SiLU(x;β)=xσ(βx)\mathrm{SiLU}(x;\beta)=x\sigma(\beta x)SiLU(x;β)=xσ(βx)

Another popular choice is GELU (Gaussian Error Linear Unit):

GELU(x)=xΦ(x)\mathrm{GELU}(x)=x\Phi(x)GELU(x)=xΦ(x)

where Φ\PhiΦ is the cdf of the standard normal distribution:

Φ(x)=P(N(0,1)≤x)=12(1+erf(x/2))\Phi(x)=P(\mathcal{N}(0,1)\leq x)=\frac{1}{2}\Big(1+\mathrm{erf}(x/\sqrt{2})\Big)Φ(x)=P(N(0,1)≤x)=21​(1+erf(x/2​))

We see that this is not a convex or monotonic function, unlike most activation functions.

We can think of GELU as a “soft” RELU since it replaces the step function with the Gaussian cdf.

See PyTorch's non-linear activations module (opens in a new tab) for an exhaustive list.

13.4.4 Residual connections

One solution to the vanishing problem is to use residual networks or ResNet. This is a feedforward model in which each layer has the form of a residual block:

Fl′(x)=Fl(x)+x\mathcal{F}_l'(\bold{x})=\mathcal{F}_l(\bold{x})+\bold{x}Fl′​(x)=Fl​(x)+x

where F\mathcal{F}F is a standard shallow non-linear mapping (e.g. linear-activation-linear).

Residual connections are often used conjointly with CNNs, but can also be used in MLPs.

A model with residual connections has the same number of parameters as without it but is easier to train. The reason is that gradient can flow directly from the output to the earlier layers.

To see this, note that the activation at the output layer can be derived as:

zL=zl+∑k=lL−1Fk(zk,θk)\bold{z}_L=\bold{z}_l+\sum_{k=l}^{L-1}\mathcal{F}_k(\bold{z}_k,\theta_k)zL​=zl​+k=l∑L−1​Fk​(zk​,θk​)

We can therefore compute the gradient of the loss wrt the parameters of the lllth layer as follows:

∂L∂θl=∂zl∂θl∂L∂zl=∂zL∂θl∂L∂zL∂zL∂zl=∂zL∂θl∂L∂zL(1+∑k=1L−1∂∂zlFk(zk,θk)=∂zL∂θl∂L∂zL+other  terms\begin{align} \frac{\partial \mathcal{L}}{\partial \theta_l} &= \frac{\partial \bold{z}_l}{\partial \theta_l} \frac{\partial \mathcal{L}}{\partial \bold{z}_l}\\ &= \frac{\partial \bold{z}_L}{\partial \theta_l}\frac{\partial \mathcal{L}}{\partial \bold{z}_L}\frac{\partial \bold{z}_L}{\partial \bold{z}_l} \\ &=\frac{\partial \bold{z}_L}{\partial \theta_l}\frac{\partial \mathcal{L}}{\partial \bold{z}_L}(1+\sum_{k=1}^{L-1} \frac{\partial}{\partial \bold{z}_l}\mathcal{F}_k(\bold{z}_k,\theta_k) \\ &=\frac{\partial \bold{z}_L}{\partial \theta_l}\frac{\partial \mathcal{L}}{\partial \bold{z}_L}+\mathrm{other\;terms} \end{align}∂θl​∂L​​=∂θl​∂zl​​∂zl​∂L​=∂θl​∂zL​​∂zL​∂L​∂zl​∂zL​​=∂θl​∂zL​​∂zL​∂L​(1+k=1∑L−1​∂zl​∂​Fk​(zk​,θk​)=∂θl​∂zL​​∂zL​∂L​+otherterms​​

We see that the gradient at layer lll depends directly on the gradient at layer LLL, independently from the network depth.

13.4.5 Parameter initialization

Since the objective function of DNNs is non-convex, the initial values of parameters can play a big role in the final solution, as well as how easy the function is to train (i.e. how well signal can flow backward and forward in the model).

13.4.5.1 Heuristic initialization scheme

It has been shown that sampling parameters from a standard normal distribution with fixed variance can lead to exploding activation or gradient.

To see why, consider a linear unit oi=∑j=1ninwijxjo_i=\sum_{j=1}^{n_{in}}w_{ij}x_joi​=∑j=1nin​​wij​xj​ where wij∼N(0,1)w_{ij}\sim \mathcal{N}(0,1)wij​∼N(0,1), E[xj]=0\mathbb{E}[x_j]=0E[xj​]=0, V[xj]=γ2\mathbb{V}[x_j]=\gamma^2V[xj​]=γ2:

E[oi]=∑j=1ninE[wijxj]=∑j=1ninE[wij]E[xj]=0V[oi]=E[oi2]−E[oi]2=∑j=1ninE[wij2xj2]−0=∑j=1ninE[wij]2E[xj2]=ninσ2γ2\begin{align} \mathbb{E}[o_i]&=\sum_{j=1}^{n_{in}}\mathbb{E}[w_{ij}x_j]=\sum_{j=1}^{n_{in}}\mathbb{E}[w_{ij}]\mathbb{E}[x_j]=0 \\ \mathbb{V}[o_i] &= \mathbb{E}[o_i^2]-\mathbb{E}[o_i]^2 =\sum_{j=1}^{n_{in}}\mathbb{E}[w_{ij}^2x_j^2]-0=\sum_{j=1}^{n_{in}}\mathbb{E}[w_{ij}]^2\mathbb{E}[x_j^2]=n_{in}\sigma^2 \gamma^2 \end{align}E[oi​]V[oi​]​=j=1∑nin​​E[wij​xj​]=j=1∑nin​​E[wij​]E[xj​]=0=E[oi2​]−E[oi​]2=j=1∑nin​​E[wij2​xj2​]−0=j=1∑nin​​E[wij​]2E[xj2​]=nin​σ2γ2​​

To keep the variance from blowing up, we need to ensure ninσ2=1n_{in}\sigma^2=1nin​σ2=1 during forward pass, where ninn_{in}nin​ is the fan-in (input connections).

When doing the backward pass, we need noutσ2=1n_{out}\sigma^2=1nout​σ2=1, where noutn_{out}nout​ is the fan-out.

To satisfy both requirements we set:

12(nin+nout)σ2=1\frac{1}{2}(n_{in}+n_{out})\sigma^2=121​(nin​+nout​)σ2=1

or equivalently:

σ2=2nin+nout\sigma^2=\frac{2}{n_{in}+n_{out}}σ2=nin​+nout​2​

This is known as Xavier initialization (use it for linear, tanh, logistic and softmax activation function).

In the special case of nin=noutn_{in}=n_{out}nin​=nout​, we use σ2=1/nin\sigma^2=1/n_{in}σ2=1/nin​, this is LeCun initialization (use it for SELU).

Finally, σ2=2/nin\sigma^2=2/n_{in}σ2=2/nin​ is Hue initialization (use it for ReLU and variants).

13.4.5.2 Data-driven initialization

We can adopt a data-driven approach to parameter initialization, like layer-sequential unit-variance (LSUV), working as follow:

  • Initialize the weights (fully connected or convolutional) using orthonormal matrices, by drawing W∼N(0,I)W\sim \mathcal{N}(0,\bold{I})W∼N(0,I) then using QR or SVD decomposition.
  • For each layer lll, compute the variance of activation across a minibatch
  • Rescale the weights as Wl=Wl/vlW_l=W_l/\sqrt{v_l}Wl​=Wl​/vl​​

This can be viewed as orthonormal initialization combined with batch normalization applied on the first mini-batch. This is faster than full-batch normalization and can work just as well.

13.4.6 Parallel training

Training large networks can be slow, and specialized hardware for matrix-matrix multiplication like graphics processing units (GPU) and tensor processing unit (TPU) can boost this process.

If we have multiple GPU, we can further speed things up.

The first approach is model parallelism, in which we partition the model across different machines. This is quite complicated since this requires tight communication between machine and we won’t discuss it further.

The second approach is data parallelism, which is much simpler as it is embarrassingly parallel. For each training step ttt:

  • We split the minibatch across KKK machines to get Dtk\mathcal{D}_t^kDtk​.
  • Each machine kkk computes its own gradient gtk=∇θL(θ,Dtk)\bold{g}_t^k=\nabla_\theta \mathcal{L}(\theta,\mathcal{D}_t^k)gtk​=∇θ​L(θ,Dtk​)
  • We sum all gradient on a central machine gt=∑k=1Kgtk\bold{g}_t=\sum_{k=1}^K \bold{g}_t^kgt​=∑k=1K​gtk​
  • We broadcast the summed gradient back to all machines, g~tk=gt\tilde{\bold{g}}_t^k=\bold{g}_tg~​tk​=gt​
  • Each machine update its parameters using θtk=θt−1k−ηtg~tk\theta_t^k=\theta_{t-1}^k-\eta_t \tilde{\bold{g}}_t^kθtk​=θt−1k​−ηt​g~​tk​

See this tutorial (opens in a new tab) for a toy implementation in Jax, and this tutorial (opens in a new tab) for a more robust solution in PyTorch.

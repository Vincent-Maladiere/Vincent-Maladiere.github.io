23.3 Shallow Graph Embedding

23.3 Shallow graph embeddings

Shallow embedding methods are transductive graph embedding methods, where the encoder maps categorical nodes IDs onto a Euclidean space through an embedding matrix.

Each node vi∈Vv_i\in Vvi​∈V has a corresponding embedding Zi∈RLZ_i\in\R^LZi​∈RL and the shallow encoder function is:

Z=ENC(ΘE)≜ΘE∈RN×LZ=\mathrm{ENC}(\Theta^E)\triangleq \Theta^E\in \R^{N\times L}Z=ENC(ΘE)≜ΘE∈RN×L

The embedding dictionary ZZZ is directly learned as model parameters.

In the unsupervised case, the embeddings are optimized to recover information about the input graph (e.g. the adjacency matrix WWW). This is similar to dimension reduction methods like PCA, but for graphs.

In the supervised case, the embedding are optimized to predict some labels, for nodes, edges and/or the whole graph.

23.3.1 Unsupervised embeddings

In the unsupervised case, we consider two main types of shallow embedding methods: distance-based and outer product-based.

Distance-based methods optimize embeddings such that nodes iii and jjj which are close on the graph (as measured by some graph distance function) are embedded in ZZZ such that the pairwise distance function d2(Zi,Zj)d_2(Z_i,Z_j)d2​(Zi​,Zj​) is small.

d2d_2d2​ can be customized, which lead to Euclidean or non-Euclidean embeddings. The decoder outputs:

W^=DEC(Z;ΘD)\hat{W}=\mathrm{DEC}(Z;\Theta^D)W^=DEC(Z;ΘD)

with W^ij=d2(Zi,Zj)\hat{W}_{ij}=d_2(Z_i,Z_j)W^ij​=d2​(Zi​,Zj​).

Pairwise dot-products compute node similarities. The decoder network can be written as:

W^=DEC(Z;ΘD)=ZZ⊤\hat{W}=\mathrm{DEC}(Z;\Theta^D)=ZZ^\topW^=DEC(Z;ΘD)=ZZ⊤

In both cases, embeddings are learned by minimizing the graph regularization loss:

LG,RECON(W,W^;Θ)=d1(s(W),W^)\mathcal{L}_\mathrm{G,RECON}(W,\hat{W};\Theta)=d_1(s(W),\hat{W})LG,RECON​(W,W^;Θ)=d1​(s(W),W^)

where s(W)s(W)s(W) is an optional transformation and d1d_1d1​ is a pairwise distance function between matrix, which don’t need to be the same form as d2d_2d2​.

23.3.3 Distance-based: non-Euclidean methods

So far, we have assumed method creating embeddings in the Euclidean space.

However, hyperbolic geometry is ideal for embedding trees and graph exhibiting a hierarchical structure.

Embedding of hierarchical graphs can be learn using the Pointcaré model of hyperbolic space. We only need to change d2d_2d2​:

d2(Zi,Zj)=arcosh(1+2∣∣Zi−Zj∣∣22∣∣1−Zi∣∣22∣∣1−Zj∣∣22)d_2(Z_i,Z_j)=\mathrm{arcosh}(1+2\frac{||Z_i-Z_j||^2_2}{||1-Z_i||^2_2||1-Z_j||^2_2})d2​(Zi​,Zj​)=arcosh(1+2∣∣1−Zi​∣∣22​∣∣1−Zj​∣∣22​∣∣Zi​−Zj​∣∣22​​)

The optimization then learns embedding that minimize the distance between connected node, and maximize the distance between disconnected nodes:

d1(W,W^)=∑i,jWijlog⁡e−W^ij∑k:Wik=0e−W^ikd_1(W,\hat{W})=\sum_{i,j}W_{ij}\log \frac{e^{-\hat{W}_{ij}}}{\sum_{k:W_{ik}=0}e^{-\hat{W}_{ik}}}d1​(W,W^)=i,j∑​Wij​log∑k:Wik​=0​e−W^ik​e−W^ij​​

where the denominator is approximated by negative sampling.

Note that since the hyperbolic space has a manifold structure, we need to make sure that the embedding stay remain on the manifold (by using Riemannian optimization techniques).

It has been shown that variant using Lorentz model of hyperbolic space provides better numerical stability.

23.3.5 Outer product-based: Skip-gram methods

Skip-gram word embeddings are optimized to predict context words given a center word. Given a sequence of words (w0,…,wT)(w_0,\dots,w_T)(w0​,…,wT​), skip-gram will optimize:

L=−∑−K≤i≤K,i≠0log⁡p(wk−i∣wk)\mathcal{L}=-\sum_{-K\leq i \leq K,i\neq 0}\log p(w_{k-i}|w_k)L=−−K≤i≤K,i=0∑​logp(wk−i​∣wk​)

for each target word wkw_kwk​.

This idea has been leveraged for graph embedding in the DeepWalk framework, since the author have proven that the frequency statistics induced by random walk in the graph is similar to that of words in natural language.

Deepwalk train node embeddings to maximize the probability of context nodes for each center node. The context are the nodes reached during a single random walk.

Each random walk starts with a nodes vi1∈Vv_{i_1}\in Vvi1​​∈V and repeatedly samples the next node uniformly at random: vij+1∈{v∈V:(vij,v)∈E}v_{i_{j+1}}\in \{v\in V:(v_{i_j},v)\in E\}vij+1​​∈{v∈V:(vij​​,v)∈E}. The walk length is a hyperparameter.

All generated random-walks (sentences) can then be encoded by a sequence model. This has been implemented by node2vec.

It is common to underlying representation to use two distinct representation for each node: one for when the node is a center and one for when it is in the context.

To present DeepWalk on the GraphEDM framework, we can set:

s(W)=Eq[(D−1W)q]  with  q∼P(Q)=Cat(1,…,Tmax⁡)s(W)=\mathbb{E}_q[(D^{-1}W)^q]\;\mathrm{with}\;q\sim P(Q)=\mathrm{Cat}(1,\dots,T_{\max})s(W)=Eq​[(D−1W)q]withq∼P(Q)=Cat(1,…,Tmax​)

Training DeeWalk is equivalent to minimizing:

LG,RECON(W,W^;Θ)=∑ilog⁡∑jexp⁡(W^ij)−∑vi∈V,vj∈Vs(W)ijW^ij\mathcal{L}_\mathrm{G,RECON}(W,\hat{W};\Theta)=\sum_i\log \sum_j \exp(\hat{W}_{ij})-\sum_{v_i\in V,v_j\in V}s(W)_{ij}\hat{W}_{ij}LG,RECON​(W,W^;Θ)=i∑​logj∑​exp(W^ij​)−vi​∈V,vj​∈V∑​s(W)ij​W^ij​

where W^=ZZ⊤\hat{W}=ZZ^\topW^=ZZ⊤ and the left term can be approximated in O(N)O(N)O(N) time by hierarchical softmax.

Skip-gram methods can be viewed as implicit matrix factorization, which can inherit benefits of efficient sparse matrix operatioons.

23.3.6 Supervised embeddings

In many applications, we have labeled data in addition to node features and graph structure.

While we can tackle supervised problem by first learning unsupervised embeddings and apply them to a supervised task, this is not the recommended workflow. Unsupervised node embeddings might not preserve important graph properties (e.g., node neighborhoods) that are most useful for a downstream task.

A number of methods combine these two steps, like label propagation (LP), a very popular algorithm for semi-supervised node classification. The encoder is a shallow model represented by a lookup table ZZZ.

LP use the label space to represent the node embedding directly (the decoder is the identity function):

y^N=DEC(Z;ΘC)=Z\hat{y}^N=\mathrm{DEC}(Z;\Theta^C)=Zy^​N=DEC(Z;ΘC)=Z

Laplacian eigenmaps are used in the regularization to enforce a smoothness on labels, which represents the assumption that neighbor nodes should have the same labels:

LG,RECON(W,W^;Θ)=∑ijWij∣∣yiN−y^jN∣∣22\mathcal{L}_\mathrm{G,RECON}(W,\hat{W};\Theta)=\sum_{ij}W_{ij}||y_i^N-\hat{y}_j^N||^2_2LG,RECON​(W,W^;Θ)=ij∑​Wij​∣∣yiN​−y^​jN​∣∣22​

LP minimizes this loss on the space of functions that take fixed values on label nodes (i.e. {y^iN=yjN:∀i  ∣  vi∈VL}\{\hat{y}_i^N=y_j^N:\forall i \;|\;v_i \in V_L\}{y^​iN​=yjN​:∀i∣vi​∈VL​} using an iterative algorithm that updates unlabeled node’s label distribution via the weighted average of its neighbors’ labels.

Label spreading (LS) is a variant of label propagation which minimizes the following function:

LG,RECON(W,W^;Θ)=∑ijWij∣∣yiNDi−y^jNDj∣∣22\mathcal{L}_{\mathrm{G,RECON}}(W,\hat{W};\Theta)=\sum_{ij}W_{ij}||\frac{y_i^N}{\sqrt{D_i}}-\frac{\hat{y}_j^N}{\sqrt{D_j}}||^2_2LG,RECON​(W,W^;Θ)=ij∑​Wij​∣∣Di​​yiN​​−Dj​​y^​jN​​∣∣22​

where Di=∑jWijD_i=\sum_{j}W_{ij}Di​=∑j​Wij​ is the degree of node viv_ivi​.

In both methods, the supervised loss is the distance between the predicted labels and the ground truth (one-hot vectors)

LSUPN(yN,y^N;Θ)=∑i∣vi∈VL∣∣yiN−y^jN∣∣22\mathcal{L}^N_{\mathrm{SUP}}(y_N,\hat{y}_N;\Theta)=\sum_{i|v_i\in V_L}||y^N_i-\hat{y}^N_j||^2_2LSUPN​(yN​,y^​N​;Θ)=i∣vi​∈VL​∑​∣∣yiN​−y^​jN​∣∣22​

These methods are expected to work well with consistent graphs, that is graph where node proximity is positively correlated with label similarity.

10.2 Binary Logistic Regression

Binary logistic regression corresponds to the following model:

p(y∣x,θ)=Ber(y∣σ(w⊤x+b))p(y|\bold{x},\theta)=\mathrm{Ber}(y|\sigma(w^\top \bold{x} + b))p(y∣x,θ)=Ber(y∣σ(w⊤x+b))

where θ=(w,b)\theta=(w,b)θ=(w,b) are the weights and bias. In other words:

p(y∣x,θ)=11+e−(w⊤x+b)p(y|\bold{x},\theta)=\frac{1}{1+e^{-(w^\top \bold{x}+b)}}p(y∣x,θ)=1+e−(w⊤x+b)1​

and we call the log odds:

a=w⊤x+b=log⁡(p1−p)a=w^\top \bold{x} +b=\log (\frac{p}{1-p})a=w⊤x+b=log(1−pp​)

10.2.2 Nonlinear classifiers

We can often make a problem linearly separable by preprocessing the input.

Let:

The resulting function is still linear, which simplifies the learning process, and we gain power by learning the parameters of ϕ(x)\phi(\bold{x})ϕ(x). We can also use polynomial expansion up to the degree KKK, the model becomes more complex and risk overfitting.

We can use SGD to solve the equation:

where the loss here is the NLL.

If we use a mini-batch of size 1, we have:

wt+t=wt−ηt∇wNLL(wt)=wt−ηt(μn−yn)xn\begin{align} w_{t+t}&=w_t-\eta_t \nabla_w\mathrm{NLL}(w_t) \\ &= w_t-\eta_t (\mu_n-y_n)x_n \end{align}wt+t​​=wt​−ηt​∇w​NLL(wt​)=wt​−ηt​(μn​−yn​)xn​​​

Since we know the objective is convex we know this will converge to the global optimum, provided we decay the learning rate appropriately.

10.2.5 Perceptron Algorithm

A perceptron is a binary classifier:

f(xn,θ)=I(w⊤xn+b>0)f(x_n,\theta)=\mathbb{I}(w^\top x_n+b >0)f(xn​,θ)=I(w⊤xn​+b>0)

This can be seen as a modified logistic regression, where the sigmoid is replaced by a Heaviside function. Since the Heaviside is not differentiable, we can’t use gradient-based optimization to fit the model.

Rosenblatt proposed a perceptron learning algorithm, where we only make updates for wrong predictions (we replaced soft probabilities with hard labels):

wt+1=wt−ηt(y^n−yn)xnw_{t+1}=w_t-\eta_t (\hat{y}_n-y_n)x_nwt+1​=wt​−ηt​(y^​n​−yn​)xn​
  • If y^n=0\hat{y}_n=0y^​n​=0 and yn=1y_n=1yn​=1, the update is wt+xnw_t+x_nwt​+xn​
  • If y^n=1\hat{y}_n=1y^​n​=1 and yn=0y_n=0yn​=0, the update is wt−xnw_t-x_nwt​−xn​

The advantage of Perceptron is that we don’t need to compute probabilities, which is useful when the label space is vast.

The disadvantage is that this method will only converge if the data is linearly separable, whereas SGD for minimizing the NLL will always converge to the globally optimal MLE.

In section 13.2, we will generalize perceptrons to nonlinear functions, enhancing their usefulness.

10.2.6 Iteratively weighted least square

Being a first-order method, SGD can be slow, especially when some directions point steeply downhill, while others have a shallow gradient.

In this situation, it can be much faster to use second-order methods that take the curvature of the space into account. We focus on the full batch setting (with NNN small) since it is harder to make second-order methods work in the stochastic setting.

The hessian is assumed to be positive-definite to ensure the update is well-defined.

Ht=∇2L(wt)=H(wt)\mathrm{H}_t= \nabla^2\mathcal{L}(w_t)=\mathrm{H}(w_t)Ht​=∇2L(wt​)=H(wt​)

If the hessian is exact, the learning rate can be set to 1:

wt+1=wt−Ht−1gt=wt−(X⊤StX)−1X⊤(y−μt)=(X⊤StX)−1X⊤Stzt\begin{align} w_{t+1}&=w_t- \mathrm{H}_t ^{-1}g_t \\ &= w_t-(X^\top S_tX)^{-1}X^\top(y-\mu_t) \\ &= (X^\top S_t X)^{-1}X^\top S_t z_t \end{align}wt+1​​=wt​−Ht−1​gt​=wt​−(X⊤St​X)−1X⊤(y−μt​)=(X⊤St​X)−1X⊤St​zt​​​

with:

zt≜Xwt+St−1(y−μt)z_t\triangleq X w_t+S^{-1}_t(y-\mu_t)zt​≜Xwt​+St−1​(y−μt​)

and:

St=diag(μt,n(1−μt,n))S_t=\mathrm{diag}(\mu_{t,n}(1-\mu_{t,n}))St​=diag(μt,n​(1−μt,n​))

Since StS_tSt​ is diagonal, we can rewrite:

The optimal weight w∗w^*w∗ is a minimizer of:

∑n=1NSt,n(zt,n−wt⊤xn)2\sum_{n=1}^NS_{t,n}(z_{t,n}-w_t^\top x_n)^2n=1∑N​St,n​(zt,n​−wt⊤​xn​)2

Fisher scoring is the same as IRLS except we replace the Hessian with its expectation i.e. we use the Fisher information matrix. Since the Fisher information matrix is independent of the data, it can be precomputed. This can be faster for problems with many parameters.

10.2.7 MAP estimation

Logistic regression can overfit when the number of parameters is high compared to the number of samples. To get this wiggly behavior, the weight values are large (up to +/-50 compared to 0.5 when K=1).

One way to reduce overfitting is to prevent weights from being so large, is performing MAP estimation using a zero mean Gaussian prior:

p(w)=N(w∣0,CI)p(w)=\mathcal{N}(w|0,C\mathrm{I})p(w)=N(w∣0,CI)

The new objectives and gradient takes the form:

PNNL(w)=NNL(w)+λw⊤w∇wPNNL(w)=g(w)+2λw∇w2PNNL(w)=H(w)+2λI\begin{align} \mathrm{PNNL}(w)&=\mathrm{NNL}(w)+\lambda w^\top w \\ \nabla_w\mathrm{PNNL}(w)&=g(w)+2\lambda w \\ \nabla_w^2\mathrm{PNNL}(w) &=\mathrm{H}(w)+2\lambda \mathrm{I} \end{align}PNNL(w)∇w​PNNL(w)∇w2​PNNL(w)​=NNL(w)+λw⊤w=g(w)+2λw=H(w)+2λI​​

10.2.8 Standardization

Using the previous isotropic prior N(w∣0,CI)\mathcal{N}(w|0,C\mathrm{I})N(w∣0,CI) assumes that all weights are of the same magnitudes, which assumes features are of the same magnitudes as well.

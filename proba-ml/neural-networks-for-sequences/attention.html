15.4 Attention

So far, we have only considered hidden activations as z=φ(Wv)z=\varphi(W\bold{v})z=φ(Wv), where WWW is a fixed set of learnable weights.

However, we can imagine a more flexible model where we have a set of mmm feature vectors V∈Rm×vV\in\mathbb{R}^{m\times v}V∈Rm×v, and the model dynamically decides which feature to use, based on the similarity between an input query q∈Rq\bold{q}\in\mathbb{R}^{q}q∈Rq and a set of keys K∈Rm×kK\in\mathbb{R}^{m\times k}K∈Rm×k.

If q\bold{q}q is most similar to ki\bold{k}_iki​, we use vi\bold{v_i}vi​. This is the basic idea behind attention mechanisms.

Initially developed for sequence models, they are now used in a broader set of tasks.

15.4.1 Attention as a soft dictionary lookup

To make this lookup operation differentiable, instead of retrieving a single value vi\bold{v}_ivi​, we compute a convex combination as follow:

Attn(q,(k1:m,v1:m))=∑i=1mαi(q,k1:m)vi∈Rv\mathrm{Attn}(\bold{q},(\bold{k}_{1:m},\bold{v}_{1:m}))=\sum_{i=1}^m \alpha_i(\bold{q},\bold{k}_{1:m})\bold{v}_i \in \mathbb{R}^{v}Attn(q,(k1:m​,v1:m​))=i=1∑m​αi​(q,k1:m​)vi​∈Rv

where αi\alpha_iαi​ is the ith attention weight, and 0<αi<10<\alpha_i<10<αi​<1, ∑iαi=1\sum_i \alpha_i=1∑i​αi​=1, computed as:

αi(q,k1:m)=Si([a(q,k1),…,a(q,km)])=exp⁡(a(q,ki))∑j=1mexp⁡(a(q,kj))\alpha_i(\bold{q},\bold{k}_{1:m})=\mathcal{S}_i([a(\bold{q},\bold{k}_1),\dots,a(\bold{q},\bold{k}_m)])=\frac{\exp(a(\bold{q},\bold{k}_i))}{\sum_{j=1}^m\exp(a(\bold{q} , \bold{k}_j))}αi​(q,k1:m​)=Si​([a(q,k1​),…,a(q,km​)])=∑j=1m​exp(a(q,kj​))exp(a(q,ki​))​

with the attention score function a(q,ki)∈Ra(\bold{q},\bold{k}_i)\in\mathbb{R}a(q,ki​)∈R.

In some cases, we want to restrict attention to a a subset of the dictionary. For example, we might pad sequences to a fix length (for efficient mini-batching), and “mask out” the padded locations. This is called masked attention.

We implement this by setting the attention score to large negative number like −106-10^6−106, so that the exponential output will be zero (this is analogous to causal convolution).

15.4.2 Kernel regression as non-parametric attention

Kernel regression is a nonparametric model of the form:

f(x)=∑i=1nαi(x,x1:n)yif(x)=\sum_{i=1}^n \alpha_i(x,x_{1:n})y_if(x)=i=1∑n​αi​(x,x1:n​)yi​

The similarity α\alphaα is computed by using a density kernel in the attention score:

Kσ(u)=12πσ2e−12σ2u2\mathcal{K}_\sigma(u)=\frac{1}{\sqrt{2\pi \sigma^2}}e^{-\frac{1}{2 \sigma^2}u^2}Kσ​(u)=2πσ2​1​e−2σ21​u2

where σ\sigmaσ is called the bandwidth. We then define a(x,xi)=Kσ(x−xi)a(x,x_i)=K_\sigma(x-x_i)a(x,xi​)=Kσ​(x−xi​).

Since the score are normalized, we can drop the 12πσ2\frac{1}{\sqrt{2\pi\sigma^2}}2πσ2​1​ term and write it:

K(x;w)=exp⁡(−w22u2)\mathcal{K}(x;w)=\exp(-\frac{w^2}{2}u^2)K(x;w)=exp(−2w2​u2)

Plugging this into the first equation gives us:

f(x)=∑i=1nSi([K(x−x1,w),…,K(x−xn,w)])yif(x)=\sum_{i=1}^n \mathcal{S}_i([\mathcal{K}(x-x_1,w),\dots,\mathcal{K}(x-x_n,w)])y_if(x)=i=1∑n​Si​([K(x−x1​,w),…,K(x−xn​,w)])yi​

We can interpret this as a form of nonparametric attention, where the queries are the test points xxx, the keys are the training points xix_ixi​ and the values are the training labels yiy_iyi​.

If we set w=1w=1w=1, we obtain the attention matrix Aji=αi(xj,x1:n)A_{ji}=\alpha_i(x_j,x_{1:n})Aji​=αi​(xj​,x1:n​) for test input jjj:

15.4.3 Parametric attention

Comparing a scalar query (test point) to each of the scalar values in the training set doesn’t scale well to large training sets, or high-dimensional inputs.

In parametric models, we have a fixed set of keys and values, and we compare keys and queries in a learned embedding space.

One general way to do it is assuming q∈Rq\bold{q}\in\mathbb{R}^qq∈Rq and k∈Rk\bold{k}\in\mathbb{R}^kk∈Rk might not have the same size, so we compare them by mapping them in a common embedding space of size hhh.

This gives us the following additional attention function:

a(q,k)=wv⊤tanh⁡(Wqq+Wkk)∈Ra(\bold{q},\bold{k})=\bold{w}_v^\top\tanh(W_q\bold{q}+W_k\bold{k})\in\mathbb{R}a(q,k)=wv⊤​tanh(Wq​q+Wk​k)∈R

with Wq∈Rh×qW_q\in\mathbb{R}^{h\times q}Wq​∈Rh×q, Wk∈Rh×kW_k\in\mathbb{R}^{h\times k}Wk​∈Rh×k

A more computationally efficient approach is to assume the keys and queries both have the same size ddd.

If we assume these to be independent random variables with mean 0 and unit variance, the mean of their inner product is 0 and their variance is ddd, since V[∑iXi]=∑iV[X]i\mathbb{V}[\sum_i X_i]=\sum_i \mathbb{V}[X]_iV[∑i​Xi​]=∑i​V[X]i​.

To ensure the variance of the inner product stays 1 regardless of the size of the inputs, we divide it by d\sqrt{d}d​.

We can define the scaled dot-product attention:

a(q,k)=q⊤kd∈Ra(\bold{q},\bold{k})=\frac{\bold{q}^\top\bold{k}}{\sqrt{d}} \in\mathbb{R}a(q,k)=d​q⊤k​∈R

In practice, we deal with minibatch of nnn vectors at a time. The attention weighted output is:

Attn(Q,K,V)=S(QK⊤d)V∈Rn×v\mathrm{Attn}(Q,K,V)=\mathcal{S}(\frac{QK^\top}{\sqrt{d}})V\in\mathbb{R}^{n\times v}Attn(Q,K,V)=S(d​QK⊤​)V∈Rn×v

with Q∈Rn×d,K∈Rm×d,V∈Rm×vQ\in\mathbb{R}^{n\times d},K\in\mathbb{R}^{m\times d},V\in\mathbb{R}^{m\times v}Q∈Rn×d,K∈Rm×d,V∈Rm×v and the softmax function applied row-wise.

15.4.4 Seq2Seq with attention

In the seq2seq model from section 15.2.3, we used a decoder in the form:

htd=f(ht−1d,yt−1,c)\bold{h}_t^d=f(\bold{h}^d_{t-1},\bold{y}_{t-1},\bold{c})htd​=f(ht−1d​,yt−1​,c)

where c\bold{c}c represents the fixed-length encoding of the input x1:T\bold{x}_{1:T}x1:T​. We usually set c=hTe\bold{c}=h^e_Tc=hTe​, the final state of the encoder (or average pooling for bidirectional RNN).

However, for tasks like machine translation, this can result in poor predictions, since the decoder doesn’t have access to the input words themselves.

We can avoid this bottleneck by allowing the output words to “look at” the input words. But which input should them look at, since each language has its own word order logic?

We can solve this problem in a differentiable way by using soft attention, as proposed by this paper (opens in a new tab). In particular, we can replace the fix context vector c\bold{c}c in the decoder with a dynamic context vector ct\bold{c}_tct​ computed as:

ct=∑i=1Tαi(ht−1d,h1:Te)hie\bold{c}_t=\sum_{i=1}^T\alpha_i(\bold{h}^d_{t-1},\bold{h}^e_{1:T})\bold{h}^e_ict​=i=1∑T​αi​(ht−1d​,h1:Te​)hie​

where the query is the hidden state of the decoder at the previous step, and both the keys and values are all the hidden state of the encoder. When the RNN has multiple hidden layers, we take the one at the top).

We then obtain the next hidden state as:

htd=f([yt−1;ct],ht−1d)\bold{h}^d_t=f([\bold{y}_{t-1};\bold{c}_t],\bold{h}_{t-1}^d)htd​=f([yt−1​;ct​],ht−1d​)

We can train this model the usual way on sentence pairs.

We can observe the attention weights computed at each step of decoding to determine which input words are used to generate the corresponding output.

Each output word yt\bold{y_t}yt​ was sampled from htd\bold{h}^d_{t}htd​ (query, vertical axis), and each line represent the attention of this decoder hidden state with all the encoder hidden states h1:Te\bold{h}^e_{1:T}h1:Te​(keys, horizontal axis).

15.4.5 Seq2Vec with attention (text classification)

We can also use attention with sequence classifiers. For example, this paper (opens in a new tab) applies an RNN classifier to predict the death of patients. It uses a set of electronic health record as input, which is a time series containing structured data as well as unstructured text (clinical note).

Attention is useful for identifying the “relevant” parts of the inputs.

15.4.6 Seq+Seq2Vec with attention (text pair classification)

Our task is now to predict whether two sentences (premises and hypothesis) are in agreement (premises entails the hypothesis), in contradiction or neutral.

This is called textual entailment or natural language inference. A standard benchmark is Stanford Natural Language Inference (SNLI) corpus, consisting in 550,000 labeled sentence pairs.

Below is a solution to this classification problem presented by this paper (opens in a new tab).

Let A=(a1,…,am)A=(\bold{a}_1,\dots,\bold{a}_m)A=(a1​,…,am​) be the premise and B=(b1,…,bn)B=(\bold{b}_1,\dots,\bold{b}_n)B=(b1​,…,bn​) by the hypothesis, with ai,bj∈RE\bold{a}_i,\bold{b}_j\in\mathbb{R}^{E}ai​,bj​∈RE the words embedding vectors.

First, each word in the premise attends to each word in the hypothesis, to compute an attention weight:

eij=f(ai)⊤f(bj)e_{ij}=f(\bold{a}_i)^\top f(\bold{b}_j)eij​=f(ai​)⊤f(bj​)

where f:RE→RDf:\mathbb{R}^E\rightarrow \R^Df:RE→RD is an MLP.

We then compute a weighted average of the matching words in the hypothesis:

βi=∑j=1nexp⁡(eij)∑k=1nexp⁡(eik)bj\beta_i=\sum_{j=1}^n \frac{\exp(e_{ij})}{\sum_{k=1}^n \exp(e_{ik})}\bold{b}_jβi​=j=1∑n​∑k=1n​exp(eik​)exp(eij​)​bj​

We compare ai\bold{a}_iai​ and βi\beta_iβi​ by mapping their concatenation to a hidden space using an MLP g:R2E→RHg:\R^{2E}\rightarrow \R^Hg:R2E→RH:

vA,i=g([ai,βi]),  i=1,…,mv_{A,i}= g([\bold{a}_i,\beta_i]),\; i=1,\dots,mvA,i​=g([ai​,βi​]),i=1,…,m

Finally, we aggregate over the comparison to get an overall similarity of premise to hypothesis:

vA=∑i=1mvA,iv_A=\sum_{i=1}^m v_{A,i}vA​=i=1∑m​vA,i​

We can similarly get an overall similarity of hypothesis to premises by computing:

αj=∑i=1mexp⁡(eij)∑k=1mexp⁡(ekj)aivB,j=g([bj,αj]), j=1,…,nvB=∑j=1nvB,j\begin{align} \alpha_j &=\sum_{i=1}^m \frac{\exp(e_{ij})}{\sum_{k=1}^m \exp(e_{kj})}\bold{a}_i \\ \bold{v}_{B,j} &=g([\bold{b}_j,\alpha_j]),\,j=1,\dots,n \\ \bold{v}_B &=\sum_{j=1}^n \bold{v}_{B,j} \end{align}αj​vB,j​vB​​=i=1∑m​∑k=1m​exp(ekj​)exp(eij​)​ai​=g([bj​,αj​]),j=1,…,n=j=1∑n​vB,j​​​

At the end, we classify the output using another MLP h:R2H→R3h: \R^{2H}\rightarrow \R^3h:R2H→R3:

y^=h([vA,vB])\hat{y}=h([\bold{v}_A,\bold{v}_B])y^​=h([vA​,vB​])

We can modify this model to learn other kinds of mappings from sentence pairs to label. For instance, in the semantic textual similarity task, we predict how semantically related two inputs are.

If we force the attention heatmap to be sparse, so that each output only attends to a single input location instead of a weighted combination of all of them, we perform hard attention.
